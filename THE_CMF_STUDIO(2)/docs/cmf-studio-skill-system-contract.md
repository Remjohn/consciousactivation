# CMF Studio Skill System Contract

## 1. Purpose

CMF Studio has two skill families:

- stable operational skills;
- JIT skill compilers.

Not all skills are JIT. JIT skill compilers are used when specialized execution must be compiled from saturation context, contrast, evidence, primitives, and failure cases.

## 2. Stable Operational Skills

Stable operational skills are repeatable procedures that agents use every day:

- registry lookup;
- source evidence review;
- consent inspection;
- command proposal;
- blocker inspection;
- receipt comparison;
- provider capability check;
- review read-model assembly.

They can be versioned as skill manifests and bound to agents.

## 3. JIT Skill Compiler Modes

JIT skill compilers are required for:

- CRAL research distillation;
- Context Premise shaping;
- Conscious Interview Brief compilation;
- narrative induction;
- interview engineering;
- expression extraction;
- contrastive prompt layers;
- anti-draft calibration;
- primitive coalition testing;
- route candidate comparison;
- Voice DNA and Emotional DNA support;
- evaluation support.

They are not only final-output writing prompts.

## 3A. Conscious Interview Brief Skill

The first monthly artifact is produced by the Conscious Interview Brief Skill. This skill reverse-engineers the old CCF trigger-first content pipeline into CCP V9/V9.1 interview engineering. It does not reduce the pipeline to a one-hop mapping from signal to question.

The old CCF lineage is treated as an intelligence stack, not a slogan:

```text
SCRE / CRAL signal discovery
-> primary signal packet
-> edge selection / pressure framing
-> coach provocation
-> authenticated response capture
-> primitive candidate generation
-> coalition survival and weighting
-> route recommendation
-> source artifact emission
```

CMF interview-first compilation then becomes:

```text
Guest Truth
+ Interviewer Resonance
+ Audience Reality / Context Premise
-> High-Identification Question Field
-> Research Field / Broad Primary Signal
-> Matrix of Edging pressure selection
-> target expression state
-> First-Line Anchor set
-> Depth Anchor
-> Interview Asset Contract
-> Complete Expression Session
-> landing evaluation
-> Expression Moment extraction
-> archetype / derivative / CMF route
```

The skill must not generate questions from a topic, content format, or blank prompt. It compiles Interview Asset Contract candidates from CRAL findings, audience conversations/comments, Guest Dossier, Audience Reality Brief, Context Premise, Audience Deep Trigger Map, Interviewer Resonance Context, Matrix of Edging, target expression states, First-Line Anchors, Depth Anchors, primitive candidates, invariant/coalition references, prior eval receipts, and hard negatives. The "question" is only one field inside the contract; the real output is the induction and extraction architecture around that question.

Its output is not a script. Its output is a set of Interview Brief asset contracts that state:

- main question;
- source pressure refs;
- Context Premise ref;
- Interviewer Resonance ref;
- Matrix and primitive refs;
- target expression states;
- First-Line Anchor options;
- Depth Anchor;
- route target: archetype, asset derivatives, CMF routes, Guest Asset Pack potential;
- expected source material;
- clip start and depth eval rules;
- landing eval targets;
- repair followups;
- intended guest reaction;
- intended extraction outcome;
- anti-centroid and unsupported-inference checks;
- downstream evaluation hints.

## 4. Required JIT Inputs

Each JIT compiler run must carry:

- source docs or evidence refs;
- transcript or guest/source artifact refs when applicable;
- Guest Dossier;
- Audience Reality Brief;
- Context Premise;
- Audience Deep Trigger Map;
- CRAL findings;
- Interviewer Resonance Context;
- audience conversation/comment refs;
- Matrix of Edging primitive candidates;
- invariant, coalition, and edge-product refs when available;
- CCF orchestration refs for signal/provocation/reaction lineage;
- route constraints;
- Brand Context Version;
- prior evaluation receipts;
- failure corpus or hard negatives;
- active primitive obligations.

## 5. Required JIT Outputs

Each run emits:

- proposals;
- contrast candidates;
- hard negatives;
- calibration report;
- evidence refs;
- primitive obligation refs;
- `SkillInvocationRecord`;
- downstream eval target hints.

JIT output cannot influence extraction, routing, or approval unless evidence grounding and anti-draft calibration pass.
