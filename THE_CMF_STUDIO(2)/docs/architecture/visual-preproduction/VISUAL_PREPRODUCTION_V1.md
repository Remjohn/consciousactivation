# CCP Visual Preproduction V1

## Definition

Visual Preproduction V1 is the shared system that transforms source truth, brand context, primitive intent, visual research, and asset intelligence into validated visual directives before any provider generation, composition lock, or render.

Its core operating law is:

```text
Recognition before creation.
```

A visual should not be generated only because it is stylish. It should first be recognizable, source-grounded, primitive-bound, and composition-ready.

## Position in the architecture

```text
Source Truth + Brand Context + Primitive Coalition
        ↓
Visual Preproduction
        ↓
Asset Intelligence requirements
        ↓
Style Route Engine
        ↓
Composition / Provider Jobs / Render
        ↓
SuperVisual / Carousel / Video / CAC / GMG / Paper-Cut
```

## What it owns

```text
Visual Researcher logic
Visual Schema
16 Elements of Visual Familiarity
Subjective Distortion Map
Storyboard Ingredients
Visual Beat Plan
PRIMAL Analysis
VAE Decoder
Anti-Cliché Gate
Constraint Gate C
T-Code / V-Code
Shot Type
Kinetic Verb
Lighting Preset
Visual Analyst Report
Storyboard Commander Verdict
Visual Preproduction Packet
```

## What it does not own

```text
asset storage
asset rights
provider execution
final image generation
final layout
video timeline rendering
carousel sequencing
publishing
```

## Depth modes

```text
lite
  simple SuperVisual, proof card, soft-rounded editorial card

standard
  premium SuperVisual, carousel slide, CAC/GMG/Paper-Cut scene

full_batch
  carousel sequence, video scene pack, motion sequence, campaign visual system
```

## Hard laws

```text
No VisualSchema without brand_context_version_id.
No silent blank 16-element Visual Familiarity checklist.
No schema claim without source-evidence status.
No provider-facing visual directive before VisualPreproductionPacket freeze when the route requires preproduction.
No full-batch Carousel or Video generation without StoryboardCommanderVerdict.
No packet freeze when Constraint Gate C has blocking violations.
No CAC/Paper-Cut/Documentary Proof directive without real-life/source-context requirements.
```
