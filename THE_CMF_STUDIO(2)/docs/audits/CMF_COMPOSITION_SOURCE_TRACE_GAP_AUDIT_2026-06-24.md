# CMF Composition Source Trace Gap Audit - 2026-06-24

## 1. Audit Purpose

This audit repairs a documentation and planning failure: the recent composition build-up was too visual-first and did not explicitly carry enough of the source intelligence from CCP V9, CCP V9.1, and the Brand Genesis / Micro-Semiotic Pipeline.

The issue is not that no specs mention those systems. Several specs already reference them. The issue is that the composition runtime layer was not yet forced to bind every visual template to:

- locked Brand Context Version;
- 64-state acting library;
- Paper-Cut avatar rig and creative libraries;
- micro-semiotic anchor library;
- Interview Asset Contract;
- Complete Expression Session;
- source Expression Moment and transcript timing;
- archetype route, asset derivative, CMF route, and four-video slot;
- primitive triad validation;
- deterministic renderer props;
- integration adapter decision receipts for open-source components.

Without those bindings, composition previews can look impressive while remaining non-compliant with CMF STUDIO doctrine.

## 3. Source Requirements That Must Govern Composition

### 3.1 Brand Genesis V3 Requirements

| Source Area | Required Product/Runtime Meaning |
|---|---|
| Brand Context Version | A composition cannot be brand-generic. It must retrieve from a locked creative state containing identity pack, acting library, Paper-Cut rig, visual constitution, micro-semiotic anchor library, motion library, SFX library, and composition preference library. |
| Complete Editing Session | Media rendering is downstream of a source Expression Session and source Expression Moment. The editing session stores asset type, core archetype, derivative, CMF route, visual style, brand ID, and Brand Context Version ID. |
| 64-State Acting Library | Acting assets are emotional-performance primitives, not random poses. Each selected human/avatar state must declare emotion, gesture family, body language, facial expression, energy level, framing, orientation, and layout bias. |
| Paper-Cut Avatar System | Paper-Cut is a rigged production system, not a flat image style. It requires head/facial expressions, eyes, brows, mouth shapes, body layers, gesture variants, rig manifest, bones, preview tests, and stop-motion jitter. |
| Paper-Cut Prop/Object Library | Props, headline strips, notes, arrows, bursts, plants, icons, stamps, mascots, and object packs are brand-scoped creative assets. |
| Micro-Semiotic Anchoring | High-identification compositions require 1-3 approved anchors that create recognition without stealing attention, stereotyping the audience, or creating legal risk. |
| Editorial 2.5D Paper-Cut Animation | Official feeling is credible, warm, handmade, premium, educational. Motion principle: paper gently coming alive. Motion must direct attention, reveal meaning, add tactile realism, or mark an emotional beat. |
| Ideogram 4 | Ideogram 4 is the Composition Director for layout, hierarchy, subject placement, text area, paper-strip arrangement, prop density, metaphor, color balance, and anchor placement. It is not final identity or final text authority. |
| GPT Image 2 | Used as onboarding asset factory for 64-state acting references, avatar sheets, expression sheets, mouth-shape sheets, prop packs, style explorations, and repair edits. |
| Flux / ComfyUI | Used for identity replacement, face/likeness refinement, local repair, paper texture harmonization, and character integration. Runtime agents may select approved references but must not alter identity conditioning. |
| Layer Extraction and Rigging | Final production needs layer manifests, editable assets, alpha/edge scores, bounding boxes, anchors, z-index, shadow behavior, and motion affordances. |
| Renderer Routing | Remotion is default production compiler; Motion Canvas handles procedural/vector explainers; Manim handles data/timeline/procedural abstractions; SCAIL-2/motion transfer is limited to memes, dancing motions, reaction clips, and recurring motion formats. |
| Render Evaluation | Each asset must score identity consistency, composition quality, style consistency, emotional accuracy, platform fit, negative space, hook strength, shareability, routeability, motion restraint, and anchor effectiveness. |
| Core JSON Chain | Composition must sit inside the full chain: BrandGenesisSession -> BrandContextVersion -> CompleteExpressionSession -> ExpressionMoment -> ArchetypeRoute -> AssetPackageSpec -> CompleteEditingSession -> SceneSpec -> CompositionPlan -> LayerManifest -> AnimationPlan -> RenderJob -> EvaluationReceipt -> ApprovalEvent -> PublishingIntent -> BrandMemoryEvent. |

### 3.2 CCP V9 Requirements

| Source Area | Required Product/Runtime Meaning |
|---|---|
| Narrative State Induction | The moat is not editing, prompting, or generic AI. The system must guide a human into an expression state using research, interviewer resonance, audience reality, Matrix of Edging, question architecture, and anchors. |
| Expression State vs Archetype | Expression states are induction modes, not asset types. Correct chain is Expression State -> Archetype Route -> Asset Contract. |
| Retained Archetype Inventory | Compositions must map to the retained inventory: storytelling, comparison/contrast, myth/scam, listicles, authority/education, humor/distortion, Conscious Reactions, and compilation containers. |
| Response Anchors | First-Line Anchors create clean clip starts. Depth Anchors prevent flat, centroid answers. Landing is evaluated, not forced. |
| Interview Asset Contract | Every production question must include content archetype, compiler family, target expression states, edge product, anchors, question, expected depth, primitives, clip start rule, landing eval targets, asset routes, and repair followups. |
| Complete Expression Session | One interview session produces many Expression Moments, asset routes, and Complete Editing Sessions. It carries dossier, resonance, audience reality, Matrix brief, asset contracts, transcript, anchor hits, primitive pass, edge products, package spec, learning, and eval receipt. |
| Three-Context Interview Engine | High-identification questions emerge from Guest Truth plus Interviewer Resonance plus Audience Reality. |
| Matrix of Edging | Matrix selects meaningful tension and protects it from polite dilution. It supplies broad primary signal, edge products, primitive candidates, coalition signatures, and anti-centroid pressure. |
| CMF Technical Update | Old input was trigger/content intent. New input is Extracted Expression Moment containing transcript segment, anchor timestamp, expression state, route, edge product, primitive coalition, asset route, visual metaphor candidates, captions, and audience identification point. |
| V9 Agent Roles | Guest Research, Audience Reality, Interviewer Pre-Induction, Matrix of Edging, Narrative State Induction, Asset Contract, Live Interview Assistant, Post-Session Extraction, CMF Asset Producer, and Evaluation agents must be traceable in pipeline receipts. |

| Source Area | Required Product/Runtime Meaning |

## 7. Audit Conclusion

The composition layer was not ready to be called source-compliant. It had important direction, but not enough mandatory binding to the full CMF documentation.

The repair is not to discard the current specs. It is to add the missing binding layer so every composition template proves:

```text
source expression -> route reason -> brand substrate -> primitive obligations -> template JSON -> timed renderer props -> eval receipt -> operator approval
```

Until TS-CMF-080 is implemented and the 24 templates are created as executable JSON plus renderer components, the current visual boards remain previsual references, not production templates.
