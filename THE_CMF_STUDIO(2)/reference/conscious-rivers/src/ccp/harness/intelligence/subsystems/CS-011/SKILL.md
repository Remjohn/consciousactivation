# CS-011 Skill

## Purpose

Apply the Peak-End Budget Allocator so CMF spends disproportionate compositional energy on the two moments memory actually keeps: the peak and the ending.

## Use This Skill When

- allocating graphics, motion, and sound complexity across scenes
- deciding where premium revision time should go
- reducing over-polish in middle beats that do not drive retrospective value

## Required Inputs

- arc positions
- predicted peak location
- ending design
- available effect and revision budget

## Decision Procedure

1. Identify the primary peak beat and the final end-state beat.
2. Allocate extra complexity budget to those beats.
3. Reduce equal-treatment bias in SETUP, CHALLENGE, and bridge scenes.
4. Return a resource map and any underfunded peak/end warnings.

## Runtime Guidance

- Memory is not democratic across scenes.
- Premium attention belongs to TURNING_POINT and VISION unless strong evidence says otherwise.
- If resources are constrained, sacrifice middle polish before peak/end clarity.
