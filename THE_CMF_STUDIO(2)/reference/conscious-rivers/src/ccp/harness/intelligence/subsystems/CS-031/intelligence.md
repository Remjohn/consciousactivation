# CS-031 - Scene Type Selector

CS-031 exists because the biological containers of the CMF arc are stable, but the cinematic vehicles that fill them must vary across tribe, inventory, and recent output. In CMF, scene choice should therefore be reasoned, not improvised.

The subsystem reads container contracts, component compatibility, available assets, recent history, and active subsystem constraints. It succeeds when the chosen component-template pair satisfies the biological job of the container while preserving novelty and feasibility. It fails when scene choice is driven by vibes, habit, or convenience.

CS-031 is the practical hinge between a valid architecture and a reusable creative library.

## Current CMF Thresholds

- choose by biological fit first
- novelty second
- preference third
- require subsystem-aware validation before final selection

## Good Example

The system replaces a no-face cinematic HOOK with an out-of-context talking-head HOOK because early retention and face-lock constraints take precedence.

## Failure Example

The system keeps selecting the same convenient scene type without checking whether it still fits the active biological and variation constraints.
