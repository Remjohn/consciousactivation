# CS-025 Skill

## Purpose

Apply the Retention Cascade subsystem to determine whether the opening 3 seconds are compressed enough to earn the viewer's permission for the rest of the video.

## Use This Skill When

- selecting a HOOK component or HOOK template
- auditing the first 3 seconds of an exported timeline
- deciding whether an intro is too slow, too cluttered, or too delayed

## Required Inputs

- first-3-second timeline slice
- ASL inside the HOOK window
- time to face
- time to primary stakes or primary signal
- opening CLS and focal clarity

## Decision Procedure

1. Measure whether the strongest signal appears early enough.
2. Measure HOOK ASL and compare it to the target band.
3. Confirm the opening does not front-load clutter or delayed explanation.
4. Return a survival score and `PASS`, `REVISE`, or `BLOCK`.

## Runtime Guidance

- Front-load signal, not ceremony.
- A logo or "today I want to talk about" opener is a retention liability.
- Treat the first 3 seconds as a compression chamber, not an introduction.

## Escalate Or Block When

- the payload arrives after the viewer's decision window
- the first 3 seconds exceed the HOOK complexity budget
- the opener uses high effort before proving relevance
