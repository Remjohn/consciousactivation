# CS-030 Skill

## Purpose

Apply the Neural Phase Router so scene parameters match the cognitive state each container is supposed to recruit.

## Use This Skill When

- validating container builds against their intended brain-state job
- checking whether a vulnerability scene behaves like vulnerability instead of hype
- routing motion, pacing, and saliency defaults from container identity

## Required Inputs

- container type
- target neural phase
- scene parameters
- emotional function

## Decision Procedure

1. Determine the neural phase implied by the container.
2. Compare motion, pacing, and saliency settings against that target.
3. Revise scenes whose formal parameters induce the wrong cognitive state.

## Runtime Guidance

- Container identity should drive parameter defaults.
- A mislabeled scene is often a misbuilt scene.
- Use neural-phase consistency as an architecture-level audit.
