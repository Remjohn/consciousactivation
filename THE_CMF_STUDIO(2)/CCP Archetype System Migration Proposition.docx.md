# CCP Archetype System Migration Proposition

## From Monolithic Prompts to Schema-Driven Agentic Content Compilation

**Document Type:** Architecture Proposition  
**Project:** Conscious Coaching Platform / Conscious Media Factory / Conscious Rivers  
**Version:** V1.0 — Archetype Contract Migration Layer  
**Status:** Proposed update for agentic environment analysis  
**Purpose:** Transform the old archetype prompt stack into a modular schema system compatible with Interview-First Expression Sessions, Narrative State Induction, Matrix of Edging, CMF render routing, and Guest Asset Pack fulfillment.

---

# I. Executive Summary

The old CCP content prompt stack should not be discarded.

It contains valuable production intelligence:

* content structures

* emotional journeys

* visual metaphors

* carousel grammars

* meme mechanisms

* research synthesis protocols

* authenticity protocols

* tribal resonance systems

* memetic evaluation criteria

* visual prompt patterns

However, the old system has a structural problem:

Each prompt tried to be the strategist, researcher, interviewer, copywriter, visual director, meme engineer, renderer, and evaluator at the same time.

That architecture does not fit the current CCP direction.

The current direction is:

Interview-first expression  
↓  
Narrative State Induction  
↓  
First-Line Anchors \+ Depth Anchors  
↓  
Expression Moment Extraction  
↓  
Archetype Contract Schema  
↓  
Asset Derivative Schema  
↓  
CMF Render Route  
↓  
Evaluation Receipt

Therefore, the old prompts should be migrated into a new modular registry architecture.

The key shift:

Old:  
Prompt generates content.

New:  
Schema defines how content should be induced, extracted, structured, routed, rendered, and evaluated.

This proposition defines how to transform the old prompts into the new CCP system.

---

# II. Core Architectural Problem

The old prompt stack was powerful but monolithic.

A file like “Conceptual Contrast” contained:

* purpose

* priming

* role instruction

* authenticity protocol

* research protocol

* variables

* template structure

* output expectations

A file like “Dopamine Cliff Carousel” contained:

* emotional journey logic

* fresh/deep research logic

* slide sequence

* audience desire mapping

* reality check mechanism

* solution framing

A file like “Validation Reaction” contained:

* reaction psychology

* tribal validation mapping

* emotional timing

* memetic trigger protocol

* authenticity logic

* shareability logic

These should not remain as single prompts.

They should be decomposed into typed objects.

The new system should separate:

Meaning structure  
≠  
Asset packaging  
≠  
Meme mechanism  
≠  
Reaction format  
≠  
Research layer  
≠  
Visual rendering mode  
≠  
Evaluation rubric

This allows agents to reason modularly instead of treating every format as a one-off prompt.

---

# III. New Registry Architecture

The new CCP archetype system should contain five registries.

CCP Archetype System  
│  
├── 1\. Core Content Archetype Schemas  
├── 2\. Asset Derivative Schemas  
├── 3\. Meme Mechanism Schemas  
├── 4\. Reaction Archetype Schemas  
└── 5\. CMF Render Mode Schemas

Each registry has a different role.

---

# IV. Registry 1 — Core Content Archetype Schemas

## Definition

A Core Content Archetype defines the **meaning structure** of a piece of content.

It answers:

* What kind of content is this?

* What human value does it deliver?

* What source material is required?

* What narrative beats must be present?

* What kind of tension does it organize?

* What expression states induce it?

* What outputs can it route into?

Core Content Archetypes are not visual formats.

They are meaning grammars.

## Examples from current and old stack

* Transformation Story

* Witness Story

* Backstory Reveal

* Confessional Turn

* Case Study Breakdown

* Comparison Breakdown

* Before vs After Contrast

* Conceptual Contrast

* Visual Timeline

* Worst Case Scenario

* Shocking Comparison

* Myth Debunk

* Scam Exposure

* Core Educator / Explainer

* Challenger / Frame Breaker

* Authority Proof Stack

* Observational Humor

* Industry Hypocrisy Exposure

## Why the old prompts matter here

Some old prompts should become Core Content Archetypes because they have distinct meaning structures.

These include:

* Conceptual Contrast

* Visual Timeline

* Worst Case Scenario

* Shocking Comparison

* Hero’s Journey

The Hero’s Journey file was mentioned but not included in the inspected uploaded batch. It should be reviewed later, but from the name and prior CCP context, it likely belongs in Core Content Archetypes as a narrative transformation schema.

# V. Registry 2 — Asset Derivative Schemas

## Definition

An Asset Derivative Schema defines how an existing expression moment or core archetype becomes a specific asset package format.

It answers:

* Is this a carousel?

* A poll?

* A tweet-style post?

* A quote card?

* A visual prompt?

* A short video?

* A challenge prompt?

* A reaction seed?

Asset Derivatives do not define the core meaning.

They define the packaging structure.

## Examples

* Dopamine Cliff Carousel

* Relief Peak Carousel

* Dilemma Poll / Would You Rather

* Persuasive Micro-Claim

* Thought Whisperer Extract

* Mirror Prompt

* Tension Poll

* Quote-to-Question

* Scene-to-Principle

* Identity Mirror

* Data Story Post

## Why this matters

The Guest Asset Pack offer requires:

* 4 videos

* 2 carousels

* 2 meme visuals

* 2 poll visuals

* reaction candidates

This cannot be solved by Core Archetypes alone.

The system needs derivative schemas that know how to transform a core insight into specific deliverables.

# VI. Registry 3 — Meme Mechanism Schemas

## Definition

A Meme Mechanism Schema defines why a meme works psychologically.

It answers:

* Is the humor based on benign violation?

* Incongruity?

* Relief?

* Superiority?

* Tribal absurdity?

* Status satire?

* Micro-contradiction?

The meme mechanism is not the same as the content topic.

It is the psychological engine of the meme.

## Examples

* Benign Violation

* Incongruity

* Relief Theory

* Superiority Theory

* Micro-Contradiction

* Tribal Absurdity

* Status Satire

## Why this matters

The old meme prompts contain valuable psychology.

They should not be reduced to “make a meme.”

They define distinct routes of humor and shareability:

* Benign Violation \= safe norm-breaking

* Incongruity \= unexpected connection

* Relief \= tension release

* Superiority \= in-group validation

* Micro-Contradiction \= compressed paradox

These should become selectable mechanisms inside the meme generation pipeline.

---

# VII. Registry 4 — Reaction Archetype Schemas

## Definition

A Reaction Archetype defines how CCP transforms a content moment into participatory response content.

It answers:

* What is being reacted to?

* What emotion should the reactor embody?

* What tribal feeling is being validated?

* What question should the audience answer?

* Is this a solo reaction, debate, vote, jury, duel, or validation moment?

## Examples

* Validation Reaction

* Solo Reaction

* Vote Then React

* Debate with Jury Mode

* Reaction Duel

* Reaction Seed

* Blind Rank Defense

* Redemption Round

## Why this matters

The interview-first system does not only generate clips.

It should generate reaction surfaces.

A powerful Claude quote can become:

* a standalone clip

* a carousel slide

* a poll visual

* a meme visual

* a validation reaction

* a debate seed

* a Power Hour prompt

Example:

Source quote:  
“Le silence d’un ami peut faire plus de bruit que la colère d’un ennemi.”

Reaction Seed:  
“Is silence betrayal, fear, or survival?”

Validation Reaction:  
“Finally someone said it: sometimes what breaks you is not the attack, but the person who knew better and still stayed silent.”

# VIII. Registry 5 — CMF Render Mode Schemas

## Definition

A CMF Render Mode Schema defines how an asset is physically produced.

It answers:

* Does this use real video?

* Paper-cut explainer?

* Animated avatar?

* Meme / motion transfer?

* Cinematic metaphor?

* Data story?

* Quiz / ranking?

* Static carousel?

* Motion carousel?

## Examples

* Personal-Brand Commentary

* Cinematic Story Commentary

* Paper-Cut Explainer

* Animated Avatar Explainer

* Living Commentary Reaction

* Conscious Reactions Editing

* Meme / Dance / Reaction

* Cinematic Metaphor

* Data Story

* Quiz / Ranking

* Carousel Static / Motion

## Why this matters

The old prompts often included visual instructions directly.

The new system should route visual production through CMF.

Instead of saying:

Generate a Ghibli-style worst-case image.

The new system says:

{  
  "cmf\_mode": "cinematic\_metaphor",  
  "visual\_style": "paper\_cutout\_stop\_motion",  
  "main\_metaphor": "memory trapped under a glass dome",  
  "emotion": "silent burden",  
  "renderer": "remotion\_or\_video\_generation\_branch"  
}

The visual prompt becomes a Scene Spec, not a loose instruction.

# IX. Migration Principle

The migration rule is:

New topic does not mean new archetype. New repeatable structure means new archetype.

Examples:

Love Across Boundaries \= edge product / theme, not archetype.  
Language as Identity Container \= edge product / theme, not archetype.  
Protective Silence \= edge product / theme, not archetype.  
Exile Memory \= theme, not archetype.

But:

Visual Timeline \= archetype because it has a repeatable temporal structure.  
Conceptual Contrast \= archetype because it has a repeatable A/B metaphor structure.  
Worst Case Scenario \= archetype because it has a repeatable fear/stakes scene structure.  
Dopamine Cliff Carousel \= derivative schema because it has a repeatable slide journey.  
Would You Rather \= derivative schema because it has a repeatable dilemma structure.  
Validation Reaction \= reaction archetype because it has a repeatable participatory reaction structure.

# XXIII. Strategic Conclusion

The old prompt stack was not wrong.

It was early.

It encoded many of the right instincts:

* contrast matters

* timelines teach transformation

* worst-case scenarios reveal stakes

* relief creates trust

* dopamine cliffs create pattern interrupts

* dilemmas drive participation

* validation reactions create tribal bonding

* memes need psychological mechanisms

* tweets are compressed conviction

* data needs emotional translation

But the old system compressed too many responsibilities into individual prompts.

The new CCP architecture separates those responsibilities into registries and schemas.

This creates:

* better evals

* cleaner routing

* reusable archetypes

* stronger CMF integration

* better interview-to-asset conversion

* deterministic production

* less prompt drift

* stronger agentic orchestration

The final doctrine:

**Old prompts become schema contracts.**  
**Schema contracts become compiler routes.**  
**Compiler routes become assets.**  
**Assets become products, challenges, webinars, Power Hours, and communities.**

The migration is not a cleanup.

It is the bridge between the old content factory and the new interview-first expression engine.
