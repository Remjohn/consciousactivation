# Implementation Sequence

This is a kernel-first build plan, not a reduced public MVP.

## Wave 1 — Contract kernel

Build and test:

- Pydantic contracts;
- JSON Schema export;
- registry loaders;
- immutable version and hash rules;
- state machines;
- event types;
- example validation.

Exit gate: all examples validate and registry references resolve.

## Wave 2 — Interview Brief procurement compiler

Build:

- Asset Portfolio Planner;
- Sequence Hypothesis Compiler;
- Expression Acquisition Plan Compiler;
- Interview Asset Contract V2 compiler;
- operator review UI.

Exit gate: a research bundle produces a fully traceable approved Interview Brief V2.

## Wave 3 — Live coverage system

Build:

- streaming transcript adapter;
- provisional ingredient detector;
- coverage matrix;
- cue suppression policy;
- PWA and Telegram cue surfaces;
- live event receipt.

Exit gate: a test session tracks ingredient coverage without breaking interview state.

## Wave 4 — Post-session ingredient inventory

Build:

- extraction;
- provenance;
- quality scoring;
- graph relations;
- inventory review;
- pickup planner.

Exit gate: a Complete Expression Session produces an approved inventory and gap report.

## Wave 5 — Sequence compiler

Build:

- pattern ranking;
- ingredient binding;
- loop compiler;
- emotional curve compiler;
- source-grounding validator;
- doctrine validator;
- operator preview and repair.

Exit gate: approved inventories compile deterministic sequence programs for all target formats.

## Wave 6 — Composition adapters

Integrate:

- production-grade video engine;
- carousel engine;
- single-image engine;
- 2D character engine;
- reaction engine.

Exit gate: one sequence program can be adapted without changing its semantic source graph.

## Wave 7 — Package and series sequencing

Build:

- Guest Asset Pack sequence planner;
- monthly publishing progression;
- fatigue and diversity evaluator;
- Publer schedule handoff;
- telemetry join.

Exit gate: a complete package is scheduled as a coherent relationship progression.

## Wave 8 — Learning and optimization

Build:

- acquisition success memory;
- sequence performance telemetry;
- DSPy optimization datasets;
- human revision analytics;
- registry recommendation updates.

Exit gate: the system improves recommendations while historical runs remain reproducible.
