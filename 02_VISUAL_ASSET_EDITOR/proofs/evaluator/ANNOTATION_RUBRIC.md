# Format 02 Evaluator Annotation Rubric

Classification: `non_production_readiness_proof`  
Profile: `character-scene-composition@1.1.0-draft`  
Authority status: proposal pending evaluation-authority review

Annotators label observable evidence, not assumed intent. Every case is reviewed against the immutable demand lineage, Activation Contract, Visual Semantic Pack, Visual Narrative Program, Feature Contracts, T/V route, Composition Intent and every wrong-reading lock.

## Ordinal labels

- `clear_pass`: direct, repeatable evidence satisfies the applicable requirement.
- `borderline`: evidence is present but materially ambiguous; send to arbitration.
- `clear_fail`: observable evidence contradicts or omits the applicable requirement.
- `uncertain`: media, provenance or evaluator capability is insufficient; never coerce to pass.
- `not_applicable`: an explicit applicability rule excludes the dimension.

These labels are not numeric thresholds.

## Required dimensions

Annotate zero-second hook, pattern-match strength, pattern-interrupt strength, viewer-role clarity, activation-direction fidelity, prediction gap, payoff, affinity, anticipation residue, anti-cliche strength, wrong-reading risk, Feature Contract compliance and delete-caption/no-text survival when applicable. A wrong-reading lock result is recorded lock by lock and cannot be replaced by an aggregate aesthetic score.

## Responsible layer and repairability

Choose the earliest authoritative layer that explains the failure. `activative_lineage`, `activation_contract`, `visual_semantics`, `visual_narrative`, `feature_contract`, `somatic_route` and upstream `composition_intent` failures require an upstream amendment. VAE-owned realization failures may use `materialization`, `materialization_pose_conditioning`, `technical`, `identity_continuity` or `temporal`. Evaluation outages are `evaluation_infrastructure` and use infrastructure retry without consuming a quality-repair round.

## Review protocol

1. Two independent annotators review each calibration case.
2. They cite spatial, temporal or contract evidence for every fail and borderline label.
3. Disagreement is retained, then adjudicated by an authorized reviewer.
4. The protected set is sealed before tuning and is never drawn from this provisional seed.
5. Identity, wrong-reading, Feature Contract and applicable no-text failures remain visible as non-compensable candidates.
