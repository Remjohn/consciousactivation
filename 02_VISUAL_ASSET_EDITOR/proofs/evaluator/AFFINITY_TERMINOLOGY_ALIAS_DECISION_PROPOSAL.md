# Affinity Terminology and Alias Decision Proposal

Classification: `non_production_readiness_proof`  
Decision status: pending product and evaluation authority

Proposed canonical term: `affinity` — the post-payoff relational pull that makes the viewer want to remain with the identity, character, world or practice represented by the asset.

Proposed alias policy: accept `human_affinity` only as an explicit, versioned legacy read alias when historical receipts used it for the same construct. Never silently rewrite, merge or infer the alias. New contracts, displays and receipts use `affinity`. If evidence shows the terms encode different constructs, reject the alias and migrate through a governed mapping with preserved original values.

This proposal does not approve terminology or thresholds.
