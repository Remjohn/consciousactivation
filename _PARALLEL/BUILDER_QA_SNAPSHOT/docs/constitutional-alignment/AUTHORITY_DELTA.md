5. validation/ID_COVERAGE_REPORT.json still reports four categories and twelve source records, while LOCAL_VERIFICATION.json reports five categories and the source register contains thirteen entries.
6. The V1.2 validation script checks FR ID order but does not compare normative shard text to registry text despite its error label; the published PASS therefore does not detect the partial wording drift.
7. The Product Constitution visual chain stops at Visual Syntax handoff and omits the amendment's required T/V somatic-route request.

These are coherence concerns rather than evidence that the validated V1.1 architecture must be discarded.
