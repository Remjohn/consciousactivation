# Non-Goals

ST-01.02 does not:

- compile Harness IR, an Atomic Harness Definition, workflow/execution plans,
  acceptance tests, or a Development Capsule;
- infer semantics, parse visual syntax, discover or ratify atomicity, evaluate
  saturation, or execute the synthetic text-normalization task;
- ingest a Format 02 corpus or use provider baselines, VAE, Delegation runtime,
  ComfyUI, GPU, VLM/evaluators, external providers, or production assets;
- support Conversational Activation, Interview Expression, ReelCast, Reaction
  Receipts, Expression Moments, consent/retention/withdrawal processing, or
  other human-reaction evidence;
- add a category, compilation target, canonical registry entry, schema,
  external dependency, database/CAS production adapter, API, CLI, Control Tower,
  external handoff, publication, certification, or production authorization;
- implement ST-02.05 or any later Story.

Format 02/reference validation and external-product tracks remain preserved but
do not constrain this category-neutral Builder Core mode.
