# Atomic Harness Pipeline

The Phase 6 Atomic Harness Pipeline is a deterministic, development-only execution and composition/media control plane. It consumes approved AIR, Interview Expression, Builder and Harness objects; compiles canonical temporal and composition programs; renders bounded source-led and static reference artifacts; evaluates actual output bytes; and prepares descendant-only repair plans without acquiring upstream semantic or VAE authority.

## Phase 6 capabilities

- canonical `VideoEditProgram` and read-only timeline projections;
- exact word-boundary EDL and source/output-time mapping;
- FFmpeg source-led talking-head rendering with bounded audio fades;
- deterministic Remotion and HyperFrames binding manifests;
- Composition IR with integer semantic BBOX geometry and Negative Space;
- PRETEXT-compatible deterministic text measurement;
- Skia binding with a deterministic reference rasterizer fallback;
- SuperVisual, Carousel PNG/PDF, and reusable animation-scene realization;
- rendered-video cut evidence, static/video reparse, independent evaluation receipts;
- responsible-layer diagnosis and bounded descendant-only repair plans.

## Boundaries

AIR owns meaning. Interview Expression owns source truth. Builder owns Harness requirements. Pipeline owns execution IR and runtime evidence. VAE owns visual asset realization. Studio is a supervisory projection. Format 02, VAE Stage 5, production authorization and certification remain false.
