## Work

1. Load and hash the AIR Constitution/PRD/Epics/Stories/20 full specs, AHP PRD/Epics/Stories/45 candidate assignments, AHP integration amendment, ownership ledger, CMF migration audit, exact source-reuse crosswalk, and current repo status.
2. Reconcile ownership conflicts. AIR owns Primitive/archetype/brand/Voice DNA/Visual DNA/Final Script/role-tension/Matrix/transfer semantic compilation. Pipeline may consume, validate, retrieve, execute, evaluate, and invalidate, but not rebuild meaning.
3. Assign exactly one disposition to every candidate specification: `ACCEPT_AS_CANONICAL`, `AMEND_TO_CURRENT_AUTHORITY`, `MERGE_INTO_CANONICAL_SPEC`, `SPLIT_FOR_IMPLEMENTATION`, `SUPERSEDE_WITH_CURRENT_SPEC`, `DEFER`, or `ARCHIVE_AS_HISTORICAL`.
4. Create canonical FR, Story, owner, and spec traceability ledgers. Every active FR has one primary owner, one primary Story, one canonical or queued full spec, one authority source, one gate, and one claim ceiling.
5. Identify every missing implementation-grade spec. Do not count assignment briefs as full specs.
6. Freeze the imported-interview reference slice from source package through semantic programs, Format 07 short, SuperVisual, animation scene package, evaluation, operator correction, HumanResolution, and replay.
7. Freeze disjoint writer lanes A–H and one-spec child-agent execution packets.
8. Initialize the Spec Quality Status Registry and Build Ledger templates from the updated Skills.
9. Create a current `SOURCE_DISPOSITION_LEDGER.yaml` before validating source completeness. Classify every named source as exactly one of:
   - `REQUIRED_AUTHORITY`
   - `REQUIRED_CURRENT_IMPLEMENTATION`
   - `REQUIRED_UNIQUE_EVIDENCE`
   - `OPTIONAL_REFERENCE`
   - `DEFERRED_REFERENCE`
   - `SUPERSEDED`

   Only the first three classes may block Prompt 03 when their bytes are unavailable. A historical assignment heading such as “Mandatory files and sources to read” does not override the current exact source-reuse crosswalk, current authority, or an attributable source-disposition decision.

   When an optional/reference-only source is unavailable:
   - create a `SOURCE_GAP_NOTICE.yaml`;
   - forbid factual claims attributed to the unavailable source;
   - retain it in the research backlog;
   - continue specification reconciliation when no unique authority or behavior depends on it.

   Reconcile `SRC-EXT-001` explicitly. The AHP exact source-reuse crosswalk classifies it as `reference_only`. If the exact PDF bytes are absent, classify it as `DEFERRED_REFERENCE` unless the operator supplies and hash-locks the exact file. It must not block `TS-AHP-007`, `TS-AHP-008`, or `TS-PM-001` unless Prompt 02 proves that a unique active requirement depends solely on that paper.
10. Validate full coverage, unique ownership, no circular authority, no active Format 02, no Stage 5 authorization, and no unresolved **required** source path.
