# Activative Intelligence Pack Compiler

Compile one governed activation field from immutable human, audience, evidence,
edge, role, reaction, and applicability inputs. Preserve every authoritative
reference and emit field-level lineage. Never invent human truth or human reaction.

## Active procedure

1. Validate every input against `contracts/input.schema.json`.
2. Resolve authority conflicts before semantic work; higher authority wins and an
   unresolved contradiction stops compilation.
3. Treat Identity DNA, Audience Context Premise, Live Premise, Resonance Map, and
   Matrix of Edging as immutable evidence-backed references.
4. Compile only the bounded activation field: edge pressure, format goal, roles,
   Activative Call constraints, desired reaction, micro-commitment, applicability,
   and wrong-reading locks.
5. Attach exact source references to every output field.
6. Mark human truth and human reaction as external; do not emit a Reaction Receipt,
   Expression Moment, identity amendment, rendered asset, or external execution.
7. Validate the result against `contracts/output.schema.json`.

## Completion criteria

- All required inputs are present, immutable, and attributable.
- Every output field has non-empty lineage.
- Identity DNA is unchanged and reference-only.
- Every applicable semantic branch has at least one wrong-reading lock.
- Every `NOT_APPLICABLE` branch has a non-empty reason.
- No provider, production, certification, visual execution, or conversational
  reaction claim is introduced.

Load branch-specific detail only from the package references named by the immutable
manifest.
