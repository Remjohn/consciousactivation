# Observability contract

Emit deterministic evidence for:

- package skill ID, version, manifest hash, and package hash;
- input contract hash and output contract hash;
- authority and source reference identities;
- field-level lineage coverage;
- applicability decisions;
- wrong-reading-lock count and disposition;
- validation outcome and typed failure code;
- human-truth externality and human-reaction externality;
- maturity and readiness ceiling;
- command identity, replay status, and invalidation ancestry when supplied by the host.

Do not emit source secrets, hidden benchmark labels, provider state, machine paths, or
invented reaction evidence.
