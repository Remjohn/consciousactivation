# PRD Addendum — Architecture-Rich Context

This addendum preserves high-value context that informs Architecture and feature specifications but would blur the product boundary if embedded directly into the main PRD.

## Builder as a meta-harness

The product is itself a governed compiler whose outputs determine the architecture, JIT Skills, contracts, evaluators, repair system, and implementation plan of future Atomic Harnesses. The quality of this Builder therefore compounds across every downstream content product.

## Visual understanding pipeline

The mandatory pre-Genesis sequence is:

```text
Locked evidence
→ deterministic specimen normalization
→ Visual Syntactic Parsing
→ composition-variable classification
→ category-adapted temporal parsing where applicable
→ BBOX observations and function hypotheses
→ cross-specimen syntax induction
→ draft sequence grammar
→ draft Activative hypotheses
→ saturation verdict
→ atomicity
→ Draft Harness Model
```

The Builder must preserve measured observation, derived function, hypothesized meaning, human decision, and generated dish content as distinct knowledge classes.

## Layered skill architecture

The product retains the original CCP JIT compilation insight but clarifies four layers:

1. **Canonical Skill** — reusable, versioned, behaviorally tested procedural capability.
2. **Harness-local Skill Adaptation** — ecological mutation for one atomic harness.
3. **Skill Composition Recipe** — parameterized plan for which capabilities, references, modules, and bindings to combine.
4. **JIT Execution Capsule** — ephemeral phase-local compiled instructions and context for one exact runtime invocation.

The original Achievement Story Design Brief is therefore not obsolete. It is a rich source for three downstream artifacts: an Archetype Constitution, Skill Composition Recipe, and Runtime Binding Schema.

## Format architecture

```text
Shared Activative Core
        ↓
Canonical Format Category
        ↓
Category-local Format Profile
        ↓
Atomic Harness Constitution
```

The five canonical categories are Short-Form Edited Video, 2D Character Animation, Carousels, Supervisuals, and Conversational Activation / Human Expression. Format 02 Minimal Coach Theatre belongs to the separate 2D Character Animation category because it depends on registry-driven character performance and continuity. The governed short-form editing profiles remain under Short-Form Edited Video, while public comment, reply/DM, ReelCast Expression, and Interview Expression remain structurally supported, uncertified conversational profiles.

## Sequencing intelligence

For video, animation, and carousels, sequencing is not export metadata. The Builder must compile a semantic and perceptual sequence program joining hidden pressure, activation direction, viewer role, parsed visual/temporal syntax, format state machine, asset or character state, pacing, transition, sonic function, prediction gap, payoff, intended reaction, evaluation, and repair.

## Observability principle

The Pi Harness Control Tower is an event-sourced read and command surface over authoritative state. It is not a second database of product truth. The governing maxim is:

> No status without evidence. No change without an event. No human action without a receipt.

## Builder workflow runtime

The Builder itself is an executable human-agent-code workflow, not one large Pi skill. The canonical runtime sequence is:

```text
Human intent and authority
→ deterministic workflow routing
→ code-owned evidence and state operations
→ phase-local agent judgment
→ independent validators and evaluators
→ targeted feedback to the responsible node
→ governed human ratification or authorization
→ Development Capsule
```

The Workflow IR is separate from the Harness IR. The Harness IR describes the harness being designed; the Workflow IR describes how the Builder performs that design. A production workflow must expose actor ownership, node contracts, conditions, retries, sandboxes, budgets, events, tests, and promotion state. Release 1 must derive this workflow from a recorded end-to-end shadow run of the selected reference harness.
