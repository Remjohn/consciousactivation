# Activative Intelligence Pack Compiler — Development Evaluation

This public development corpus tests whether the bounded compiler preserves governed
identity, audience reality, edge, role, desired reaction, micro-commitment,
wrong-reading locks and exact lineage without inventing human truth or reaction.

The evaluator is deterministic and non-compensable: any hard gate yields
`insufficient_evidence`. A clean case may reach only `development_validated`.
`shadow_ready` and `certified` are recognized maturity vocabulary but are forbidden
campaign outcomes. No production threshold, protected label or certification claim is
defined here.

Fixtures extend `BASE_CASE.json`; extension is deterministic replacement/recursive
mapping merge. Independent public-development labels adjudicate semantic quality.
Code-owned checks independently enforce authority, evidence, lineage, identity,
audience, reaction and wrong-reading boundaries.
