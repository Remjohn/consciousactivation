# Acceptance Criteria

- Given exact governed semantic inputs, when the portable skill compiles an Activative Intelligence Pack, then every output field retains lineage and no human truth or reaction is invented.
- Given the generic deterministic fixture, when skill necessity is evaluated, then no real skill is required and the prior zero-skill result is unchanged.
- Given Activative mode, when necessity alternatives are evaluated, then the exact `activative_intelligence_pack_compiler@1.0.0` is required.
- Given package and evaluator assets, when validation runs, then missing anchors, wrong-reading locks, authority boundaries, evaluator evidence, or immutable pins fail closed.
- Given identical inputs, when compilation and validation repeat in a fresh process, then canonical bytes and hashes are identical.
- Given development evidence, when maturity is reported, then it is at most `development_validated`; production and certification remain false.
