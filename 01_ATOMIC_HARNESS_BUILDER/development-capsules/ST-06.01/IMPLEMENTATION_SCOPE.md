# Implementation Scope

Implement an immutable category-binding decision and receipt, validate the exact
five-category registry, enforce one category for Activative manifests, preserve a
justified `NOT_APPLICABLE` generic branch, and project the binding into portable
non-production Atomic Harness Definitions.

The implementation may add a category command service, deterministic in-memory
command repository, observations, and focused tests. It may add the category field
to the existing Activative operator-manifest contract and update its repository-owned
fixture. It must not compile category-local profiles, syntax, sequencing, runtime,
evaluation, repair, migration, conversational feedback, or external handoffs.
