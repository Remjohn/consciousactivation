# Acceptance Criteria

1. Given the governed category registry, when it is loaded, then exactly the five
   canonical IDs, names, owners, registry version, and constitutional authority are
   preserved.
2. Given an Activative Harness with complete rich lineage and locks, when one governed
   category is selected, then an immutable deterministic structural binding and receipt
   are produced with `Activation First`, `Visual Syntax First`, and uncertified status.
3. Given the conversational category, when it is bound structurally, then it is accepted
   as the fifth category without executing conversational behavior or claiming certification.
4. Given no, multiple, unsupported, or flattened category ownership for an Activative
   Harness, when binding is attempted, then it fails closed without partial state.
5. Given missing semantic lineage, wrong-reading locks, authority, or registry integrity,
   when binding is attempted, then HG-015-compatible rejection evidence is emitted.
6. Given a generic non-Activative task, when compiled, then category applicability is
   explicitly `NOT_APPLICABLE` with a governed justification.
7. Given identical inputs and a repeated command, when compilation is replayed, then
   byte-identical output and the original receipt are returned; conflicting reuse fails.
8. Given any successful output, when inspected, then production readiness and
   certification are false and no external-product behavior is present.
