# Security and Authority Boundaries

Deterministic Builder code may validate the active Builder-owned definition under the existing run authority. It may not change constitutional, human-ratified, target-profile, certification or external-product authority.

The validator has no network, provider, VAE, Delegation, GPU, database, workflow-runtime or publication authority. It consumes no credentials and copies no external schema. External compatibility remains unevaluated in this mode.

Only a new governed definition version can change validated product meaning. Only explicit human product authority may change validation policy, certification scope or activate the external compatibility branch. Unauthorized actors, stale authority references and local override attempts fail closed.
