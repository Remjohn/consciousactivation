Implement one deterministic compiler from the exact active synthetic Builder Core lineage to a typed immutable `AtomicHarnessDefinition`.

The definition must include the source profile and Source Lock; atomic boundary and human ratification; Draft Harness Model; exact Harness IR; deterministic artifacts; constitutional precedence; capability ownership; responsibility modules; Phase Graph as a non-executing plan; accepted internal handoff; Minimum Complete Context; empty-registry snapshot; no-skill-necessity decision; input/output contract; compiled acceptance-test declarations; authority/provenance; compatibility; and explicit nonproduction/noncertification markers.

The application service validates every active identity/hash/version before a single atomic in-memory development/test commit, attaches the definition to the run, emits deterministic observations and a receipt, supports idempotent replay/checkpoint resume, and propagates upstream invalidation without mutating history.

Implementation is bounded to the exact source/test paths in `ALLOWED_FILE_SCOPE.yaml`. No schema or external dependency may be added.
