# Security and authority boundaries

Only registered deterministic Builder code may compile and attach the graph. Human authority remains the source of ratification and policy approval; code validates but cannot replace it. Agents, evaluators, and external actors cannot commit governed state. Non-code capability owners are representable only with explicit authority and evidence; no such owner is activated by the synthetic fixture.

The capsule input is exact-path and hash-pinned. Arbitrary paths, network access, external discovery, environment-dependent values, secrets, and unpinned policies fail closed. The Builder owns only the ownership contract. It does not execute capabilities, assign external-product authority, resolve skills, or absorb VAE/Delegation behavior.
