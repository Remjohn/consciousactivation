# Security and Authority Boundaries

The authorized Builder application actor may consume and validate the pinned empty registry and emit the bounded snapshot and receipt. It may not modify the policy, fixture, schema, validation receipt, predecessor artifacts, Constitution, PRD, planning baseline, or shared contracts.

Deterministic Builder code owns validation, hashing, canonical ordering, exact capability coverage, immutable identity, replay, idempotency, and invalidation. Humans retain authority for new canonical capabilities, stable promotion, deprecation, and high-plasticity change. Independent evaluators retain authority for maturity evidence. No human or evaluator action is invoked because the registry is empty.

Undeclared skill use, tool/network/secret grants, dynamic discovery, hidden orchestration, external runtime calls, authority escalation, stale pins, and same-version mutation fail closed. Code-owned capabilities are not skills and must not be granted agent autonomy.

Delegation and VAE contracts and runtime behavior remain externally owned and are not dependencies of this Story. The synthetic snapshot is non-production, non-certified, and cannot authorize execution or publication.
