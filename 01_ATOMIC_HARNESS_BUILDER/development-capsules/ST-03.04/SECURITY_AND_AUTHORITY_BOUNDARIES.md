# Security and authority boundaries

HarnessIR is immutable input and the sole product-semantic authority. Generated artifacts are subordinate views and may not write, promote, repair, or reinterpret IR values. Deterministic Builder code owns rendering, canonicalization, hashing, consistency, drift detection, atomic commit, replay, and receipts. Existing human ratification remains attributable but no new human decision is required. Agents/evaluators may inspect or propose typed upstream changes only; they cannot author generated truth.

The bounded compiler uses standard-library pure functions and the in-memory development/test adapter. It has no network, provider, model, GPU, secret, external repository, source-write, arbitrary output-path, subprocess, or production publication access. Artifact paths are a closed canonical inventory; traversal, absolute paths, casefold collisions, extras, binary payloads, secrets, external URLs/references, and executable content fail closed.

The OpenSpec, dashboard, graph, skill, evaluation, repair, and implementation-ticket outputs are data projections only. They do not authorize or implement external tools, Control Tower, Workflow IR, skills, evaluations, repairs, implementation, VAE, Delegation, category products, or certification.
