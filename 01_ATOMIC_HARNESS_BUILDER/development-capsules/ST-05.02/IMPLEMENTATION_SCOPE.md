# Implementation Scope

Extend the existing ST-05.01 skill-registry domain and application seam to run one deterministic necessity test against the exact active synthetic zero-skill snapshot.

For each of the five governed capabilities, verify target-failure evidence and evaluate the capsule-defined alternatives in order. The exact accepted result is that deterministic Builder code already supplies each capability. Emit one immutable aggregate `NO_NEW_SKILL_REQUIRED` decision, one receipt, one run reference, and deterministic observations.

The decision must record `NOT_APPLICABLE_NO_GAP` for the Skill Design Brief. The contract must fail closed if a future governed `NEW_SKILL_REQUIRED` or material-adaptation branch lacks a complete typed brief and human authority, but this Story may not activate that branch.

Support identical-command replay, conflicting-payload rejection, upstream invalidation, historical reproduction, and atomic failure/clean retry. No registry, policy, schema, planning, external contract, runtime, API, UI, database, or production system changes are permitted during implementation.
