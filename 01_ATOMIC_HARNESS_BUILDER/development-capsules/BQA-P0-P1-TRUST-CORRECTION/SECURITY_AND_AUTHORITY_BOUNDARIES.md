# Security and Authority Boundaries

Authority precedence is Constitution V1.1, Builder PRD V1.2 plus its authoritative
amendment, confirmed Stories and Builder-first overlay, accepted TS/ADRs, then this capsule.
The capsule cannot amend or redefine higher authority.

Deterministic code reconstructs governed meaning, verifies hashes, actor identity and
expected versions, commits state and issues supplemental receipts. Missing, forged,
stale, superseded or unauthorized evidence fails before state mutation.

Evidence inputs are read-only and immutable. No network, credential, external repository,
external runtime or shared-schema ownership is authorized. External-target compatibility
stays `NOT_EVALUATED_EXTERNAL_TARGET_BRANCH`.

The transaction boundary must produce exactly one winner, no partial event/receipt/
idempotency/checkpoint state and a precise committed/degraded-observability outcome.
ZIP descriptors must be tied to the exact verified bytes.

Original PASS receipts remain immutable. The standing Full Builder campaign authority is
suspended until the explicit bounded constitutional-stop override is provided.
