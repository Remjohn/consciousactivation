# Acceptance Criteria

1. Given an admitted governed category/profile, frozen Shared Activative Core, rich
   object references, and hash-valid ST-06.03 evidence gate, when the authorized
   steward compiles, then one immutable category-native syntax artifact and one
   Activative sequence program preserve exact source lineage and meaning.
2. Given each of the five categories, when syntax compiles, then its spatial, temporal,
   reading-order, and conversational structures remain substrate-native and never use
   another category as a cosmetic alias.
3. Given conversational structural profiles, when compiled, then turn relationships,
   Activative Calls, reaction/expression references, landing, close, and commitment
   structures remain explicit, structural, non-production, and uncertified.
4. Given a generic non-Activative task, when applicability is evaluated, then a
   justified explicit `NOT_APPLICABLE` result is produced without category invention.
5. Given generic parser output, sparse tokens without rich references, missing syntax
   evidence, unsupported cross-category syntax, altered lineage, an Activation First
   violation, false certification, or unauthorized authority, when compilation is
   attempted, then it fails closed with zero partial state.
6. Given identical governed inputs or a repeated command, when compilation is replayed,
   then byte-identical artifacts and the original receipt result; changed input creates
   a new identity and conflicting command reuse fails.
7. Given any output, when inspected, then Visual Syntax First is recorded only as the
   development law, Activation First remains the runtime law, and production readiness
   and certification remain false.
