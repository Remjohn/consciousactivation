# Security and authority boundaries

Only registered deterministic Builder code with an active grant may compile or commit minimum-context state. Humans retain constitutional and policy authority; agents, evaluators, providers, external actors, context text, and conversation history cannot ratify references, raise budgets, weaken exclusions, change influence boundaries, or acquire write authority.

All resources come from an explicit allowlist. Ghost references, unregistered pointer targets, forbidden runtime references, must-not-influence violations, path escape, hash drift, and stale or invalidated handoffs fail closed. Required overflow cannot be waived by local configuration; a new authoritative input or authorized upstream version is required.

No raw resource payload, personal data, secret, prompt, machine path, or external-product content enters identity-bearing observations. The implementation owns no shared schema, external runtime, network transport, or production persistence.
