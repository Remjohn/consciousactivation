# Security and authority boundaries

Human authority confirmed the planning amendment and the limited BD-010 policy. That confirmation did not authorize code changes. Only the exact branch authorization phrase may open implementation.

The implementation must preserve the completed lifecycle's typed actor and transition checks. Fixture identity, target, proof classification, manifest hash, and empty-registry hash are authority-bearing inputs and must be verified before state creation. Denials must be fail-closed and leave no authoritative success event.

The synthetic fixture is repository-owned test evidence, not user content, Human Reaction material, a category constitution, a reference corpus, or production data. HD-006 and HD-007 are therefore inactive for this branch; their conversational and real-profile authority remains unchanged.

Delegation schemas and runtime remain externally owned. This branch has no direct Delegation contract dependency and must not fork them. VAE realization behavior remains externally owned and must not be implemented. Format 02 authority and evidence remain outside the branch and unchanged.

Any attempt to widen the target, register a category/profile, call an external skill/runtime/provider, publish an output, or change a pinned authority artifact is an authority violation and a hard stop.
