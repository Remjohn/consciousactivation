# V2.1 Brownfield Delta

## Retain and integrate

- Python package and CLI foundation;
- source profiles, source inspection, directory and ZIP reading;
- evidence and specimen indexing;
- saturation artifacts and source locking;
- atomicity statuses;
- guided and provisional Genesis modes;
- decision graph, ratification, status, and cascade locking;
- OpenSpec generation conventions;
- readiness receipts and implementation handoff concepts;
- Pi project and skill integration;
- existing automated tests as the migration baseline.

## Extend through the next architecture

- target-specific source and lifecycle profiles;
- Visual Syntactic Parsing and category-adapted temporal parsing;
- one typed Harness IR and artifact compiler;
- explicit capability, module, phase, context, and contract graphs;
- canonical skill registry, adaptations, recipes, and deterministic capsule compiler;
- reference-loading and Minimum Complete Context policies;
- behavioral evaluations, protected benchmarks, hard gates, and downstream results;
- event-sourced Pi Control Tower;
- Repair and Invalidation Graph;
- prototype and implementation authorization;
- four governed category constitutions;
- traceable Development Capsule.

## Known correction areas

- distinguish executable package isolation from knowledge-corpus self-containment;
- add portable source discovery and target-specific readiness diagnostics;
- prevent embedded references from being present but unregistered;
- eliminate independently authored duplicate truth across artifacts;
- move readiness beyond field presence and document completeness;
- preserve exact evaluated skill and capsule identity;
- limit general production claims to passed benchmark targets.

## Migration method

```text
Inventory current behavior
→ lock baseline fixtures
→ define one delta
→ add failing tests or benchmark cases
→ implement the minimum correction
→ compile the reference harness through baseline and candidate
→ compare behavior and downstream effect
→ retain, adapt, deprecate, or reject the change
```
