# Epic N — Outcome-Centered Title

**Epic outcome:** What the Harness Architect, maintainer, reviewer, or implementation team can accomplish after this epic.

**Requirements covered:** FR-..., NFR-..., Architecture-..., UX-...

**Dependencies:** Prior epics only.

## Story N.M — Action-Oriented Title

As a **[user type]**,  
I want **[complete vertical capability]**,  
so that **[observable value]**.

**Requirements:** FR-..., NFR-...

**Contracts and seams:** ...

**Given** [precondition]  
**When** [action]  
**Then** [expected outcome]  
**And** [event, receipt, error, or edge behavior]

### Story quality checks

- completable in one development-agent context;
- depends only on prior stories;
- delivers independently testable behavior;
- includes error and edge criteria;
- maps to every requirement it claims;
- identifies feature technical specification where required.
