# Feature Index

The product is organized into 16 behavioral feature shards aligned to the 16 locked delegation decisions.

- [F01 — Governed Protocol Boundary and Deterministic Boundary Services](./F01-protocol-boundary.md) — Independent products can delegate visual work through an enforced boundary without creating a third creative authority.

- [F02 — Visual Asset Demand Ownership, Immutability, and Authority](./F02-demand-ownership.md) — The Content Harness can express and evolve visual intent without production systems silently changing it.

- [F03 — Immutable Contract Family and Common Delegation Envelope](./F03-contract-family.md) — Every cross-product interaction is unambiguous, attributable, versioned, and independently validatable.

- [F04 — Stable External Lifecycle and Deterministic Projection](./F04-external-lifecycle.md) — Both products and operators can understand the shared commitment without coupling to internal production nodes.

- [F05 — Demand Supersession, Impact Analysis, and Selective Invalidation](./F05-supersession-invalidation.md) — An updated demand can replace an in-flight or completed version without discarding unaffected work or promoting stale results.

- [F06 — Production Acceptance and Downstream Consumption Acknowledgement](./F06-result-acknowledgement.md) — A production-certified asset enters composition only when it is still valid for current downstream state.

- [F07 — Budget Authorization, Allocation, Escalation, and Cost Receipts](./F07-budget-authority.md) — The requester controls maximum spend and policy purpose while the VAE autonomously optimizes production within that authority.

- [F08 — Cancellation, Deadlines, Safe Checkpointing, and Race Resolution](./F08-cancellation-deadlines.md) — Obsolete delegated work stops safely, preserves valuable evidence, and cannot produce a stale consumable result.

- [F09 — Protocol Failure Taxonomy, Responsibility, and Recovery Semantics](./F09-failure-taxonomy.md) — Every unsuccessful condition routes to the correct owner and recovery path without blind retries or cross-product confusion.

- [F10 — Delegation Sets, Shared Continuity, Dependencies, and Group Evaluation](./F10-delegation-sets.md) — Related assets can be produced independently while preserving scene-level identity, continuity, geometry, and completion guarantees.

- [F11 — Semantic Compatibility, Negotiation, Adapters, Migration, and Deprecation](./F11-compatibility-migration.md) — Independently released products interoperate only when required meaning and behavior can be preserved safely.

- [F12 — Typed Amendment Proposals and Authority-Governed Resolution](./F12-amendment-protocol.md) — Feasibility evidence can lead to precise, reviewable changes without the VAE silently relaxing the demand.

- [F13 — Post-Completion Invalidation, Revocation, Supersession, and Replacement](./F13-post-completion-governance.md) — Previously accepted assets remain historically reproducible while current and future consumption responds correctly to new evidence.

- [F14 — Principal Identity, Message Integrity, Replay Protection, and Audit Chain](./F14-trust-integrity.md) — Only authorized systems can create state changes, and every accepted action can be verified and reconstructed.

- [F15 — Delegation Observability, SLOs, Conformance, and Resilience](./F15-observability-conformance.md) — Operators and release gates can prove that cross-product delegation is healthy, compatible, secure, and recoverable.

- [F16 — Delegation Readiness, Development Capsule, and Production Certification](./F16-implementation-readiness.md) — Implementation begins only when the shared boundary is complete, testable, architecture-preserving, and proven across both products.
