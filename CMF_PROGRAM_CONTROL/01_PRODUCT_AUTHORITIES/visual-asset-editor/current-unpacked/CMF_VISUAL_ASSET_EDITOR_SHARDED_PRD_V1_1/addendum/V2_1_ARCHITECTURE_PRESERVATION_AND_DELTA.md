# V2.1 Architecture Preservation and Product Delta

## Preserved V2.1 foundation

The V2.1 source document `CCP_CMF_ATOMIC_HARNESS_SPEC_BUILDER_V2_1/docs/15_VISUAL_ASSET_EDITOR_ARCHITECTURE.md` defines:

- an immutable Activative Visual Asset Program;
- a Resolved Activative Visual Asset Program;
- semantic sovereignty;
- research, strategy, materialization, generation, geometry, and asset-memory responsibilities;
- seven authority lanes;
- approved-vault and real-reference priority;
- two-stage geometry;
- separation between Content Harness BBOX intent and editor image-conditioned recommendations.

The new PRD preserves these principles through the Architecture Preservation Contract.

## Deliberate product changes

### Autonomous operation

V2.1’s authority lanes become dynamically activated responsibilities inside an autonomous event-sourced workflow.

### No standalone Rights Analyst

The Rights and Provenance Analyst is removed as a routine reasoning lane. The product retains deterministic source class, provenance, origin reference, allowed-source policy, and production lineage.

### Typed Visual Production Plan IR

The editor gains a provider-neutral canonical plan between demand and ComfyUI/provider execution.

### Visual Compute Fabric

The product gains immutable Docker runtime profiles, GPU scheduling, local/cloud execution, APIs, model/LoRA mounts, health, recovery, and cost governance.

### Independent VLM quality loop

Evaluation becomes a product subsystem with family/context-specific profiles, composition simulation, recurrence reasoning, causal repair, evaluator certification, and three-round quality limits.

### Visual Asset Memory and steering

Memory gains rendered syntax usage, contextual recurrence, smart multimodal retrieval, CMF-OKF projection, and validated Steering Recipes.

### Release 1 reference proof

Format 02 Minimal Coach Theatre becomes the first real end-to-end production certification target.

## Removed assumptions

The PRD does not assume:

- one fixed seven-agent chain per demand;
- one permanent ComfyUI server;
- generation-first production;
- manual operator candidate approval;
- a universal asset evaluator;
- usage count as visual fatigue;
- OKF or vector storage as canonical operational truth.
