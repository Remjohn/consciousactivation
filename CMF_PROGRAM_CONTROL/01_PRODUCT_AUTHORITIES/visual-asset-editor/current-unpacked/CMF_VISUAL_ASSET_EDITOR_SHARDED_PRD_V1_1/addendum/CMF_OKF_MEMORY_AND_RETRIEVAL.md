# CMF-OKF Memory and Smart Retrieval

## Decision

Use Open Knowledge Format internally as a portable, versioned, human- and agent-readable **knowledge projection**. Do not use it as the source of live workflow, contract, asset-lifecycle, queue, budget, registry, or event truth.

## Layered memory

```text
immutable evidence and assets
→ typed operational state
→ multimodal indexes and typed knowledge graph
→ CMF-OKF projection
→ JIT retrieval compiler
```

## Retrieval pipeline

1. Resolve node responsibility and authority.
2. Filter by lifecycle, category, format, family, role, failure, compatibility, and validity.
3. Traverse typed graph relationships.
4. Run lexical, semantic, image, composition and syntax-fingerprint retrieval.
5. VLM-rerank against demand, composition, failure and preservation.
6. Include relevant contradictions, exceptions, superseding knowledge and known failures.
7. Compile Minimum Complete Context.
8. Emit retrieval receipt.
