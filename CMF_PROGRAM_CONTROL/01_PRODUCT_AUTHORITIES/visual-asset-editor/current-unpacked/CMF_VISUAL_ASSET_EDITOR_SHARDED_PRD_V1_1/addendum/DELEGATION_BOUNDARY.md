# Content Harness ↔ Visual Asset Editor Delegation Boundary

## Content Harness owns

- semantic intent;
- Activative function;
- viewer-state and sequence role;
- visual-syntax role;
- composition intent and tolerance;
- identity and continuity requirements;
- wrong-reading locks;
- output acceptance requirements;
- authority to approve demand amendments.

## Visual Asset Editor owns

- resolution strategy;
- asset reuse/retrieval/extraction/transformation/composite/generation/animation;
- workflow, model, LoRA, control and compute selection;
- candidate portfolio;
- asset and composition evaluation;
- targeted repair;
- asset promotion;
- geometry, masks, crops and delivery variants;
- production and budget receipts.

## Shared contract families

- Visual Asset Demand
- submission and idempotency receipt
- Visual Asset Event
- Constraint Conflict and Amendment Proposal
- Composition Geometry
- Visual Quality Evaluation
- Visual Repair Contract
- Asset Result Contract
- compatibility manifest

## Separate PRD requirement

The future Delegation PRD must finalize:

- shared schema authority;
- field-level responsibility;
- compatibility and deprecation;
- request amendment and cancellation;
- event delivery guarantees;
- caller authentication/authorization;
- error and terminal-state taxonomy;
- cross-product integration tests;
- downgrade and migration policy.

This Visual Asset Editor PRD contains representative fixtures so Architecture can proceed, but does not appropriate shared schema ownership.
