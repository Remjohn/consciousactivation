# Release 1 Reference Slice — Format 02 Minimal Coach Theatre

## Purpose

This directory defines the product-level reference fixtures required to prove the CMF Visual Asset Editor through one real Atomic Harness integration in the **2D Character Animation** category.

The slice is not a final implementation. It is the governed bridge from the PRD to Architecture, benchmark design, representative delegation contracts, and the future Development Capsule.

## Source visual laws

The Visual Syntax Builder’s Format 02 doctrine establishes:

```text
Maximum three active elements:
1. Guide character/body/gesture
2. Text field/paper card/written thought
3. Meaning object/proof/diagram/icon
```

Hard laws:

- 9:16 vertical;
- no lip sync;
- large negative space;
- deterministic text;
- prop carrying requires attachment points;
- avoid default centered front-facing gaze;
- Remotion-first final runtime.

## Release 1 product boundary

The Visual Asset Editor owns:

- registry-driven character identity, pose, expression, gesture, gaze, props, environment/scene plates;
- ComfyUI/local-model generation and transforms;
- transparent character cutouts;
- continuity and recurrence evidence;
- masks, BBOXes, safe zones, gaze and crop geometry;
- independent VLM evaluation and targeted repair;
- immutable asset/result contracts.

The Format 02 Atomic Harness and downstream Remotion runtime retain:

- sequence and scene grammar;
- deterministic text;
- final timing and composition;
- final activation and viewer-state program;
- final use of character, text field, and meaning object.

## Required proof

A real demand must be submitted through the asynchronous service, executed on one local/self-hosted and one cloud Docker GPU profile, evaluated, repaired when necessary, promoted, and consumed in a real Format 02 composition without routine manual ComfyUI work.
