# Builder V1.2 Constitutional Alignment Amendment

**Amends:** `CMF_ATOMIC_HARNESS_BUILDER_NEXT_SHARDED_PRD_V1_1`  
**Aligned package:** `CMF_ATOMIC_HARNESS_BUILDER_NEXT_SHARDED_PRD_V1_2`  
**Authority:** CCP Activative Intelligence & Visual Narrative Constitution v1.1.0  
**Status:** Binding amendment for Architecture, technical specifications, and implementation readiness

## Purpose

This amendment preserves the validated Builder architecture while correcting the category and semantic coverage required by the highest-order constitution. It does not replace Harness IR, the workflow runtime, JIT skills, Control Tower, Repair Graph, Development Capsule, or the three compilation targets.

## Binding corrections

### 1. Constitutional precedence

The Builder must pin the Activative Intelligence Constitution and treat it as controlling doctrine whenever a local category, feature, contract, or implementation decision governs Activative or visual meaning.

### 2. Fifth canonical category

The canonical category registry now includes:

```text
conversational_activation_expression
```

Initial profiles are:

```text
public_comment
reply_dm
reelcast_expression
interview_expression
```

This category owns Activative Call sequencing, Human Reaction capture, Expression Moment extraction, micro-commitments, close logic, and Identity DNA amendment proposals. It does not own downstream editing or visual production.

### 3. Shared Activative Core contract

The Shared Activative Core must preserve references and frozen versions for:

```text
Coach or Guest Identity DNA
Audience Context Premise
Coach/Guest–Audience Resonance Map
Matrix of Edging
Activative Intelligence Pack
activation directions
roles
stance
identity urges
participation design
intended reaction
micro-commitment
wrong-reading locks
evaluation contract
```

Sparse execution tokens are allowed only when they retain references to the rich authoritative objects.

### 4. Conversational runtime chain

A conversational harness must be able to compile:

```text
Activative Intelligence Pack
→ Activative Call sequence
→ Reaction Receipt
→ Expression Moment Inventory
→ post-expression Activative recompilation
→ final Content Type Contracts
```

### 5. Visual runtime chain

A visual harness must be able to compile:

```text
Activative Intelligence Pack
→ Visual Semantic Pack
→ Semiotic MCDA receipt
→ Visual Narrative Program
→ format harness
→ feature contracts
→ Composition Asset Pack
→ Visual Syntax handoff
→ T/V somatic route request
```

### 6. Dual-order law

`Visual Syntax First` remains mandatory for harness-development evidence analysis. Runtime asset production remains `Activation First`. Architecture and technical specifications must document both orders explicitly.

### 7. Wrong-reading locks

Every generative, composited, restyled, or semantically transformative visual contract must contain at least one dominant wrong-reading lock. Missing locks block readiness.

### 8. Evaluation completeness

Compiled harness evaluations must support role clarity, pattern matching, pattern interruption, prediction, payoff, affinity, anticipation residue, anti-cliché strength, no-text survival where applicable, and wrong-reading rejection.

## Affected base requirements

This release updates the meaning of `FR-137`, `FR-139`, `FR-145`, `FR-146`, and `FR-147` without changing their stable identifiers. `D031` remains historical decision evidence but is constitutionally expanded from four to five initial categories.

## Implementation acceptance tests

1. An Interview Expression candidate no longer blocks as an unsupported category.
2. A compiled interview harness exposes Activative Call, Reaction Receipt, Expression Moment, and Identity DNA proposal contracts.
3. A visual target cannot compile a demand that omits constitutional semantic lineage or mandatory wrong-reading locks.
4. Architecture distinguishes discovery-time Visual Syntax First from runtime Activation First.
5. Category migration produces impact analysis for existing four visual categories without changing their IDs or profiles.

## Non-goals

- Moving interview execution into the Builder.
- Moving ComfyUI or visual production into the Builder.
- Replacing the validated category constitutions for existing visual formats.
- Reopening the original 33 decisions beyond this explicit constitutional amendment.
