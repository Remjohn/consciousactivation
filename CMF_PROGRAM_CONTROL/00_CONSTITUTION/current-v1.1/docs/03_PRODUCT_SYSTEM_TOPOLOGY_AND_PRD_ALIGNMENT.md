# CMF Product System Topology and PRD Alignment

**Version:** 1.1.0  
**Status:** Binding explanatory architecture  
**Updated:** 2026-07-14

## What we are building

CMF is not one content generator and not one universal harness. It is a constitutional system that discovers human and audience pressure, compiles Activative Intelligence, captures or activates authentic expression, and routes that expression into separately governed production systems.

```text
Research and evidence
→ Coach / Guest Identity DNA
→ Audience Context Premise
→ Coach / Guest–Audience Resonance
→ Matrix of Edging
→ Activation Compiler
→ Activative Intelligence Pack
```

The pack then branches.

```text
Conversational branch
Activative Call
→ human reply / interview answer
→ Reaction Receipt
→ Expression Moment
→ post-expression recompilation
→ final Content Type Contracts

Visual branch
Visual Semantic Pack
→ Semiotic MCDA
→ Visual Narrative Program
→ format harness
→ feature contracts
→ Composition Asset Pack
→ Visual Syntax and exact composition
→ visual production and evaluation
```

## Product responsibilities

### Activative Intelligence constitutional hub

Owns highest-order doctrine, ontology, semantic sequence, cross-product invariants, and amendment precedence.

### CMF Atomic Harness Builder

Owns harness-development compilation. It turns evidence and ratified product decisions into canonical Harness IR, graphs, skills, workflow specifications, tests, Development Capsules, and implementation authorization. It must support both visual categories and the Conversational Activation / Human Expression category.

### Interview Expression / ReelCast Harness

Owns Guest Context intake, research compilation, Activative Interview Briefs, Interview Asset Contracts, operator-led expression sessions, transcript-grounded Reaction Receipts and Expression Moments, Identity DNA amendment proposals, and post-transcript Content Type Contracts. It does not edit or render final content.

### Atomic visual content harnesses

Own format-specific selection, sequencing, visual semantics, visual narrative, composition requirements, final content contracts, and downstream Visual Asset Demands.

### Delegation Protocol

Owns preservation of immutable, versioned contract meaning and authority between Content Harnesses and the Visual Asset Editor. It does not invent, compress away, or reinterpret semantic fields.

### Visual Asset Editor

Owns asset-resolution planning and execution: retrieval, transformation, generation, compositing, ComfyUI workflows, models, LoRAs, compute, candidate portfolios, evaluation, repair, asset lineage, and delivery. It may not reconstruct missing Activative or visual-narrative intelligence from a generic prompt.

### Visual Syntax and composition runtime

Own exact BBOX + WHY, hierarchy, attention path, spacing, crop, text placement, runtime composition, and final render integration. It does not choose upstream recognition intent or edge pressure.

## Constitutional integration rules

1. Rich upstream intelligence remains addressable even when sparse tokens are compiled for execution.
2. Every generated or semantically transformed visual demand has non-empty wrong-reading locks.
3. The Visual Asset Editor receives recognition, narrative, feature, and somatic context; it does not guess these from `semantic_intent` alone.
4. Interview and comments are full conversational harness surfaces, not unsupported exceptions to visual categories.
5. Expression Moment lineage is preserved into final content and visual contracts.
6. Evaluation measures activation, not merely clarity, beauty, or technical validity.
7. The first failed responsible layer is repaired before downstream polish.
