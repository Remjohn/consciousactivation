## Identity and version law

Each canonical object has:

- stable object ID;
- version;
- content hash;
- lifecycle state;
- producer;
- authority;
- provenance;
- supersession and invalidation relationships.

Sparse runtime tokens are not independent truth. They preserve refs to rich objects.
