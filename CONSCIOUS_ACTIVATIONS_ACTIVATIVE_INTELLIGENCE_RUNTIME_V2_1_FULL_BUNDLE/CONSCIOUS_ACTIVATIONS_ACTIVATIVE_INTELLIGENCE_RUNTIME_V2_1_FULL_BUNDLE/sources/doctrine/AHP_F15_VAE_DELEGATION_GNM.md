# F15 — Visual Asset Editor, Delegation, and GNM Boundary Integration

## 1. Product claim and user outcome

**User outcome:** The Pipeline requests visual ingredients through typed boundaries and consumes measured VAE results while VAE owns production methods such as SAM3, Lucida, layered providers, ComfyUI, and GNM.

Preserve product sovereignty from source-grounded demand through evaluated Asset Result and usage acknowledgement. This feature exists to convert doctrine and source evidence into behavior that can be implemented, tested, denied, replayed, and audited. It does not grant implementation, provider, model-training, publication, production, or certification authority by itself.

## 2. Product boundary and ownership

Pipeline owns demand/result orchestration; VAE owns visual production methods; Delegation owns cross-product lifecycle.

The feature must preserve the current product topology. Builder supplies the immutable Atomic Harness; Pipeline executes its declared nodes; Interview Expression owns source expression preparation and approval; VAE owns visual production; Delegation owns cross-product envelopes; Studio projects state and emits typed operator commands; Program Control owns cross-product authority and release truth. External renderers and models are embodiments, not sources of meaning.

## 3. Canonical objects and state

**Cross-cutting Activative law:** Content activates when it gives the viewer a psychological role inside a tension. Content-bearing work inherits the approved archetype coalition, Primitive Coalition Contract, Final Script, Source Fidelity, Negative Space, Edge Integrity / Anti-Centroid, and Brand/Voice/Visual context when applicable.

The feature consumes current, versioned upstream records and emits typed outputs that remain linked to the active Harness, source package where applicable, Workflow Node, JIT Capsule, execution binding, evaluator, and receipt. Canonical records use stable serialization and immutable identities. A UI projection, model response, provider job, renderer workspace, or local cache cannot become accepted product state until its contract and authority gates pass.

Core invariants for this feature are:

- Pipeline does not implement VAE providers.
- GNM supplies geometry, not human truth.
- Demand/result identity and evaluation gate continuation.

## 5. Workflow integration

Validated upstream state → feature behavior → typed output and receipt → downstream handoff → evaluation, invalidation, and replay.

Every handoff is typed. A downstream node begins only after the producer output passes the declared contract and evidence gates. Failures are attributed to source, knowledge, retrieval, Skill, recipe, Programmed Model, tool, runtime, product, evaluator, or human-policy layers before repair or escalation.

## 6. Functional Requirements

### FR-085 — Emit immutable source-grounded Visual Asset Demands through Delegation

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** When a source-backed content job lacks or must repair a visual ingredient, the Pipeline shall emit an immutable Visual Asset Demand through Delegation containing the exact source lineage, semantic and composition role, asset family, geometry need, permitted variation, preservation locks, budget, deadline, and evaluation contract.

**Trigger and preconditions:** A Workflow Node proves that an accepted composition or edit requires an unavailable, incompatible, or failed visual ingredient.

**Required output or state transition:** A versioned Delegation request linked to the source package, Expression Moment or evidence object, Harness, node, and composition/timeline program.

**Authority and invariants:**
- Pipeline does not plan VAE production strategy.
- The demand cannot mutate source meaning.
- Source-derived identity/expression references remain attributable.

**Acceptance scenarios**

- **Primary success:** Given a Carousel slide requiring a source-grounded portrait cutout, when the asset is unavailable, then the demand includes the exact keyframe/source-time reference and intended composition region.
- **Adversarial or denial case:** Given an agent that decides to generate a decorative image without an accepted role or demand, when admission runs, then Delegation is not invoked.
- **Evidence required:** Demand hash, lineage graph, authority check, budget reservation, and Delegation submission receipt.

**Governing decisions:** D004, D028, D030, D044, D050  
**Exact source references:** SRC-CUR-014, SRC-CUR-016, SRC-CUR-022, SRC-AM-002, SRC-INT-001


### FR-086 — Consume VAE Asset Results with image-conditioned geometry

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** The Pipeline shall consume accepted VAE Asset Results only after validating request/result identity, source and derivative lineage, asset-family scope, measured geometry, masks, gaze or motion, depth/layers, validated crops, negative-space regions, limitations, evaluation evidence, and downstream compatibility.

**Trigger and preconditions:** Delegation returns a candidate or accepted Asset Result.

**Required output or state transition:** A validated production-asset reference and geometry pack eligible for the requesting Composition or VideoEditProgram.

**Authority and invariants:**
- An asset result cannot silently change the demand.
- Limitations remain visible downstream.
- The producing model cannot self-approve.

**Acceptance scenarios**

- **Primary success:** Given a returned portrait derivative, when validation passes, then the Pipeline can place it using measured face/subject BBOXes and validated crop data without rerunning the producer.
- **Adversarial or denial case:** Given a visually attractive asset with missing demand identity or evaluator evidence, when continuation is requested, then the Pipeline blocks the downstream node.
- **Evidence required:** Demand/result match report, geometry validation, evaluator receipts, artifact hashes, and acknowledgement/usage receipt.

**Governing decisions:** D004, D028, D030, D044, D050  
**Exact source references:** SRC-CUR-014, SRC-CUR-016, SRC-CUR-018, SRC-CUR-022


### FR-087 — Keep SAM3 execution inside the Visual Asset Editor

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** SAM3 segmentation and tracking shall be implemented, versioned, evaluated, and operated by the Visual Asset Editor behind its provider and capability interfaces; the Pipeline owns only demand, result, health, and receipt integration.

**Trigger and preconditions:** A VAE route requires concept-conditioned segmentation or tracking.

**Required output or state transition:** VAE-owned masks, tracks, geometry evidence, model/runtime identities, and Asset Result references.

**Authority and invariants:**
- No SAM3 worker is implemented inside Pipeline core.
- Pipeline does not claim VAE provider capability.
- Provider output remains independently evaluated.

**Acceptance scenarios**

- **Primary success:** Given a VAE plan selecting SAM3, when the worker succeeds, then the Pipeline receives only the contract-defined result and evidence.
- **Adversarial or denial case:** Given a Pipeline node attempting to call SAM3 directly to bypass VAE planning, when ownership validation runs, then the action is denied.
- **Evidence required:** Product-boundary test, VAE capability receipt, Delegation result, and Pipeline consumer conformance.

**Governing decisions:** D004, D050  
**Exact source references:** SRC-CUR-013, SRC-CUR-015, SRC-CUR-016, SRC-MIG-001


### FR-088 — Keep Lucida matting evaluation inside the Visual Asset Editor

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** Lucida or any replacement matting provider may be evaluated and, if authorized, executed only within the VAE capability registry and production plan; the Pipeline shall not own provider selection or matting semantics.

**Trigger and preconditions:** The VAE determines that alpha refinement is required for an asset family and route.

**Required output or state transition:** VAE-owned matte artifacts, benchmark/evaluation evidence, limitations, and Asset Result bindings.

**Authority and invariants:**
- License and model evidence must be approved.
- Provider choice remains VAE-owned.
- Pipeline sees only typed result status and artifacts.

**Acceptance scenarios**

- **Primary success:** Given an approved Lucida capability and a typography cutout demand, when VAE executes, then matte evidence is packaged with the result.
- **Adversarial or denial case:** Given an unreviewed model or license, when a production plan tries to bind it, then VAE blocks before execution and Pipeline receives a typed blocker.
- **Evidence required:** Capability registry state, benchmark report, dependency adoption review, output hashes, and result receipt.

**Governing decisions:** D004, D050  
**Exact source references:** SRC-CUR-013, SRC-CUR-015, SRC-CUR-016, SRC-MIG-001


### FR-089 — Keep layered visual generation and GNM behind VAE capability routing

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** Layered image providers, ComfyUI workflows, and Google GNM facial-geometry references shall remain VAE-owned capabilities selected by the Visual Production Plan; they are not Pipeline, Interview Expression, or identity-authority components.

**Trigger and preconditions:** A Visual Asset Demand requires layer-separated generation, controlled facial geometry, expression reference, pose/gaze reference, or another specialist visual route.

**Required output or state transition:** VAE plan decisions, bounded provider artifacts, geometry/reference passes, evaluation, and Asset Result lineage.

**Authority and invariants:**
- GNM does not infer human identity or emotional truth.
- Real-person identity comes from approved source references and fitting evidence, not demographic sampling.
- Provider outputs cannot approve themselves.

**Acceptance scenarios**

- **Primary success:** Given an approved source-backed Expression Moment and demand for a controlled head-pose reference, when VAE selects GNM, then the resulting mesh/render references preserve source identity lineage and are labeled geometric controls.
- **Adversarial or denial case:** Given a request to generate a real coach identity from GNM demographic labels, when validation runs, then the route is prohibited.
- **Evidence required:** VAE route receipt, source-reference binding, GNM/version identity, geometry outputs, fairness/limitation acknowledgement, and independent evaluation.

**Governing decisions:** D004, D050  
**Exact source references:** SRC-CUR-015, SRC-CUR-016, SRC-EXT-026, SRC-AM-002, SRC-MIG-001


### FR-090 — Validate Asset Results and downstream usage before continuation

**Lifecycle:** `AMENDED_V1_1`  
**Normative requirement:** The Pipeline shall resume composition only after the VAE result passes demand identity, authority, source lineage, geometry, evaluation, artifact integrity, supersession, and usage-acknowledgement checks.

**Trigger and preconditions:** A VAE result is proposed for use by a static, temporal, or animation derivative.

**Required output or state transition:** An accepted production asset binding, downstream usage record, invalidation dependencies, or typed rejection.

**Authority and invariants:**
- Result-ready is not completed until downstream acknowledgement where required.
- Rejected assets remain historical.
- Supersession invalidates only affected descendants.

**Acceptance scenarios**

- **Primary success:** Given an evaluated result with complete geometry, when the requesting program acknowledges exact use, then the Pipeline resumes and records the consumption relationship.
- **Adversarial or denial case:** Given a superseded or revoked asset, when a cached composition attempts to use it, then eligibility fails and only affected descendants are invalidated.
- **Evidence required:** Validation report, usage acknowledgement, asset/version lineage, and selective invalidation receipt.

**Governing decisions:** D004, D028, D030, D044, D050  
**Exact source references:** SRC-CUR-014, SRC-CUR-018, SRC-CUR-019, SRC-CUR-022


## 7. Execution, model, tool, and human responsibilities

Deterministic services own mechanically decidable facts. Programmed Models or bounded Agent Programs may propose or compile behavior only within their evidence-backed claim and node-local tool grants. Independent evaluators inspect judgment dimensions. Humans own new meaning, identity interpretation, unresolved taste, operator source authority, promotion, and publication authority where assigned. The producing model or runtime cannot accept its own output.

When the feature uses a Hunter, Analyst, Composer, or Commander role, the role is an explicit workflow responsibility. It may be implemented by code, a small model, a larger model, or a human. The role name never hides the actual actor or expand its authority.

## 8. Validation and quality gates

Validation must prove both the intended result and the process that produced it. The minimum feature evidence includes Delegation conformance, demand/result lineage, VAE capability receipts, geometry validation, and downstream acknowledgement. Mechanical tests run before expensive model or visual evaluation. Judgment evaluators are calibrated to the exact category and context. Every denial names the cause, owner, and next admissible action. Every accepted mutation emits an immutable receipt and updates only dependent state.

## 9. Risks, mitigations, and stop conditions

| Risk | Concrete consequence | Mitigation |
|---|---|---|
| Authority drift | A predecessor, external project, UI, model, or renderer is treated as current product authority. | Require exact source disposition, current authority validation, and product-boundary tests. |
| False completion | A synthetic URI, dry-run, unverified command, or model self-report is promoted as a real result. | Require actual files, hashes, independent validation, and artifact-class labels. |
| Over-broad automation | A model or agent gains tools, context, or authority beyond the current node. | Use JIT context, node-local tool grants, sandboxes, and explicit escalation. |
| Hidden mutation | Accepted meaning, source, program, or human preference changes without a versioned record. | Use immutable versions, typed commands, receipts, and selective invalidation. |
| GNM identity misuse | Generic identity sampling or expression labels are treated as real-person identity or emotional truth. | Require source-backed fitting/references, VAE ownership, explicit geometric-control labels, and route prohibition tests. |

**Stop conditions**

- The behavior would create a second semantic or canonical state owner.
- A required source, authority, contract, primitive coalition, archetype, Final Script, runtime, evaluator, or human decision is unavailable or unverified.
- The implementation would treat migration evidence or an external project as current authority.
- The change would imply real output, production readiness, or certification beyond available evidence.
