# Implementation Sequence

## Gate A — Constitutional and contract foundation

Execute `TS-AIR-001` through `TS-AIR-008`, `TS-AIR-012`, `TS-AIR-018`, and `TS-AIR-019` after ratification. These establish epistemology, source context, portfolios, Primitive and archetype compilation, brand/Voice/Visual DNA, planned programs, source packages, HumanResolution, failure attribution, JIT role capsules, and exact handoffs.

## Gate B — Vertical runtime and integration

Execute `TS-AIR-009` through `TS-AIR-017` as dependency-ready vertical slices. Begin with deterministic fixtures and imported source packages, then add bounded live source-activation pilots and derivative handoffs.

## Gate C — Programmed Model and evidence promotion

Execute `TS-AIR-020` only after the controlled responsibility has a stable contract, deterministic or stronger-model baseline, corpus, evaluation, fallback, HumanResolution capture, and evidence-specific authorization.

## Streaming factory law

Accepted specs release implementation lanes continuously. Do not wait for every spec to finish, and do not let an implementation lane invent missing contracts while coding. Every lane receives one Development Capsule with exact sources, path ownership, tests, and claim ceiling.
