# Atomic Harness Builder V1.3 Contract Amendment

The Builder remains the compiler of AtomicHarnessDefinition. It does not become the Activative Intelligence Runtime.

The definition and binding requirements must be able to declare:

- required activation domains and epistemic-state constraints;
- required Planned, Observed, Campaign, Relationship, Derivative, and Transfer object references;
- Primitive Family, Primitive Binding, coalition, signature, Edge Product, and evaluation dependencies;
- psychological role and tension requirements;
- archetype coalition, Brand Context, Voice DNA, Visual DNA, RSCS, CCV, SDA, SFL, and Final Script dependencies;
- reusable 2D Animation Scene Package obligations;
- AIR, Interview Expression, Pipeline, VAE, Delegation, Studio, and evaluator product handoffs;
- capability claims for bounded Programmed Models without representing them as semantic authority.

Existing structural `activative_intelligence_pack_compiler` capability should be renamed or documented as an Activative Contract Compiler when necessary to prevent confusion with the semantic runtime.
