# AHP PRD V1.3 — Activative Intelligence Runtime Integration Amendment

## Purpose

The Atomic Harness Pipeline continues to own execution, composition, editing, runtime binding, receipts, evaluation control, and selective repair. It consumes V2.1 semantic programs and must not duplicate their compilation.

## New integration feature: F31 — Activative Semantic Program Handoff

1. The Pipeline shall consume exact Planned, Observed, Campaign, Relationship, Derivative, Primitive Coalition, Archetype Coalition, Final Script, and Activation Transfer objects declared by the active Harness.
2. Planned, observed, inferred, operator-confirmed, rejected, and superseded states shall remain distinct in JIT context and receipts.
3. Composition shall not begin until the applicable psychological role, tension, Primitive Coalition Contract, Edge Product, archetype coalition, Guest Voice DNA Final Script, and operator approval are present.
4. Every derivative shall preserve Activation Transfer invariants and source lineage through final artifact evaluation.
5. Every eligible Final Script shall yield a reusable 2D Animation Scene Package manifest; only required scenes are fully rendered.
6. Evaluation, audience observations, candidate outcomes, and HumanResolutionEpisodes shall return upstream as evidence without live semantic mutation.

## Existing feature amendments

- F02 Harness intake: accept and validate semantic dependency references.
- F04 JIT: retrieve exact activation, Primitive, archetype, brand, Voice/Visual DNA, and transfer objects.
- F06 Skills/Recipes: recognize Primitive coalitions as the Recipe-building authority.
- F09–F13 static and temporal production: enforce Final Script and Composition-Before-Editing gates.
- F16 evaluation: emit Primitive and Activation Transfer receipts alongside visual/technical evaluation.
- F17 learning: HumanResolution capture remains automatic; semantic promotion remains upstream and scoped.
- F25 derivatives: reusable 2D animation scenes become a default package obligation, while Format 02 remains deferred.
