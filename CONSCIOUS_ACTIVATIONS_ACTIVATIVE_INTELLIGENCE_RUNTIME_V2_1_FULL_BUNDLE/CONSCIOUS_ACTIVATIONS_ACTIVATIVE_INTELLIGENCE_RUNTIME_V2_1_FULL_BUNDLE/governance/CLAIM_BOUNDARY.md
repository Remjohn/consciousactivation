# Claim Boundary

This package supports these claims only:

- V2.1 doctrine, PRD, requirements, Epics, Stories, Tech Specs, contracts, inventories, and amendments are represented in one source-locked bundle.
- Every one of 120 Functional Requirements has one primary Story and one primary Tech Spec.
- Exact supplied Primitive and archetype evidence snapshots are inventoried and hash-addressed.
- Local schema and invariant tests can validate the included structural examples.

It does not support claims that:

- Program Control has ratified this product boundary;
- a production Activative Intelligence Runtime has been implemented;
- real source activation, human reaction, audience activation, relationship progression, or Activation Transfer Fidelity has been proven;
- cross-product runtime integration exists;
- Programmed Models outperform approved baselines;
- the system is production-ready or certified.
