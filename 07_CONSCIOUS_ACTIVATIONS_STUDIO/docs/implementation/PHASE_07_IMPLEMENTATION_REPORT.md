## Implemented boundaries

The implementation is a deterministic development foundation. It compiles only bounded, validated commands. Unknown natural-language requests require clarification instead of guessed edits. Named Steering Recipes may compile known taste corrections into exact operations. Semantic-authority changes are routed to AIR.

The timeline is read-only and derived from the canonical `VideoEditProgram`. Direct manipulation produces the same typed command class as natural-language requests. Selective reruns include only changed nodes and graph descendants.

Every accepted or rejected human resolution can be persisted in an append-only hash-chained ledger and indexed as scoped programming material. No live model weights, canonical Skills, Steering Recipes or doctrine are modified automatically.

Ship decisions require an eligible campaign state, explicit publication authority, publication policy, artifact evidence, evaluation evidence and no unresolved exceptions. SHADOW mode is always denied publication.

The predecessor timeline panel was not restored as canonical state.
