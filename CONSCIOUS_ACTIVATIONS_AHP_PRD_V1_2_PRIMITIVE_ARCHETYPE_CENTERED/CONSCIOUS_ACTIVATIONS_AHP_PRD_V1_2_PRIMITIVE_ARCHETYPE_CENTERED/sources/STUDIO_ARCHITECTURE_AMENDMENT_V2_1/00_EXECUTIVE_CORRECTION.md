# Conscious Activations Studio Architecture Amendment V2.1

## Governing corrections

1. **Format 02 is not in the active build queue.**  
   Its Harness does not yet exist. Existing Format 02 specifications remain historical
   and future-use inputs, but no active implementation or Studio screen may claim an
   executable Format 02 path until the Atomic Harness Builder emits and validates the
   current Format 02 Harness.

2. **Every human resolution is automatically programming material.**  
   Every approval, rejection, correction, parameter adjustment, candidate selection,
   revision request, manual timeline move, and typed explanation emits an immutable
   `HumanResolutionEpisode`. The episode is immediately indexed as scoped programming
   material for retrieval, evaluation, supervised learning, preference learning,
   hard-negative construction, repair learning, and future Programmed Model training.

3. **The operator is a creative supervisor, not the primary editor.**  
   Atomic Harnesses execute from order to artifact. The Studio exposes progress,
   candidates, evidence, and exceptions. The operator supplies initial seeds and taste,
   inspects when desired, and corrects course through typed requests.

4. **The existing VideoTimelineWorkbench timeline is not the canonical timeline UI.**  
   Reuse its route, shell, API integration patterns, styling, review concepts, and
   product context only where useful. Replace the nonfunctional timeline panel with an
   adopted timeline foundation selected through a bounded technical and license
   evaluation.

5. **One Studio shell, several category-native Studios.**  
   Do not deploy a separate complete application per Category. Build one Conscious
   Activations Studio with shared workspace, campaign, run, review, evidence, and
   artifact infrastructure. Register category-native Studio modules behind one shell.

6. **Dokploy may deploy the Studio on Google Cloud or AWS.**  
   Dokploy is infrastructure deployment and operations machinery. It does not own
   campaign workflow law. Supabase remains the application control plane; external
   CPU/GPU/render workers execute bounded jobs.

## Corrected product promise

```text
Select workspace and project
→ provide content seed, objective, source material, and taste direction
→ select Atomic Harness or Asset Package Spec
→ launch campaign
→ Harness conducts generation from start to finish
→ evaluators and deterministic checks police execution
→ Studio interrupts only for configured review gates or genuine exceptions
→ operator requests a correction when needed
→ correction is compiled into exact typed tool operations
→ Harness reruns only the affected work
→ accepted artifact ships
→ every human resolution becomes programming material
```
