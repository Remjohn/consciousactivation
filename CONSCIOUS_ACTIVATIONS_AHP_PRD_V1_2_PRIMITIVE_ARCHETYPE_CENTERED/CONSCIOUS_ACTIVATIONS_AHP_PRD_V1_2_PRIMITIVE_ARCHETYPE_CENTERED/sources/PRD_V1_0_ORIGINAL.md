# Vision and Product Promise

## Vision

Conscious Activations should turn a canonical Atomic Harness into a reliable production system that can automate real editing and composition tools with much greater precision than a generic creative agent.

## Product promise

> Given a valid, authorized `AtomicHarnessDefinition` and `HarnessExecutionBindingManifest`, the Atomic Harness Pipeline executes every declared Workflow Node through its eligible deterministic module, Programmed Model, Agent Program, human authority, independent product, or runtime embodiment; produces static, temporal, character, conversational-derivative and delegated outputs; independently evaluates and locally repairs them; and emits complete replayable receipts without changing the Harness meaning.

## Core operating flow

```text
Activative Intelligence and human expression authority
→ Atomic Harness Builder
→ AtomicHarnessDefinition
→ HarnessExecutionBindingManifest
→ governed retrieval
→ JIT Execution Capsules
→ Workflow Nodes
→ composition/editing programs
→ Skia / Remotion / HyperFrames / FFmpeg / VAE
→ independent evaluation
→ local repair or human escalation
→ final assets and learning evidence
```

The Pipeline is the governed execution and editing-automation layer. It is intended to automate OpenMontage-class toolchains while preserving current Conscious Activations doctrine, Categories and product boundaries.

# Users, Jobs, and Journeys

## Primary users

- Conscious Content Operator and creative-policy authority.
- Harness Architect and Builder operator.
- Pipeline operator and reliability engineer.
- Model Program engineer and evaluator.
- VAE and runtime adapter engineer.
- Human approver with a recorded bounded authority scope.

## Core jobs

- Execute a compiled Harness without manually reconstructing its workflow.
- Automate editing and composition while preserving declared intent.
- Use small specialized models where they demonstrably improve precision, speed or consistency.
- Retrieve only the internal doctrine, Skills, recipes and precedents applicable to the current decision.
- Render through the correct toolchain and verify the actual output and process.
- Learn from accepted, rejected and repaired production episodes without uncontrolled live self-modification.

## Journeys

### UJ-01 — Execute a compiled Harness

An operator supplies a portable AtomicHarnessDefinition and approved binding; the Pipeline verifies, schedules and executes the run.

### UJ-02 — Create Identity Anchor + Claim static composition

A static Harness retrieves one Skill and eligible Steering Recipes, compiles operations, renders candidates in Skia, evaluates and accepts or repairs.

### UJ-03 — Produce a Carousel sequence

The Pipeline compiles slide roles, swipe order, continuity and static render packages after current profile ratification.

### UJ-04 — Produce a SuperVisual

The Pipeline compiles a single-frame visual argument, validates feed legibility and renders a Skia master plus variants.

### UJ-05 — Automate an edited video

The Pipeline selects source clips, compiles Timeline IR and executes FFmpeg, Remotion or HyperFrames according to the binding.

### UJ-06 — Execute Format 02 character performance

The Pipeline compiles the 13-registry CharacterPerformanceProgram and realizes it through an approved temporal runtime.

### UJ-07 — Request a missing visual ingredient

The Pipeline emits a Visual Asset Demand through Delegation, receives an evaluated VAE result and resumes composition.

### UJ-08 — Diagnose and repair a local failure

An independent evaluator emits a typed diagnosis; the Pipeline reruns only declared affected units.

### UJ-09 — Promote a Programmed Model claim

A model team registers training lineage, benchmark evidence, advantage profile, shadow results and obtains human promotion authority.

### UJ-10 — Resolve human creative authority

The Pipeline pauses at an explicit human node, presents evidence and alternatives, records the decision scope and resumes.

### UJ-11 — Retrieve internal knowledge precisely

A node receives an authority-filtered JIT Capsule containing the governing Skill, eligible recipes, precedents and required contradiction.

### UJ-12 — Contain a runtime regression

Canary drift suspends an affected claim, routes to known-good fallback, preserves incident lineage and restores only with evidence.

# Doctrine and Product Principles

1. **Activation First at runtime.** The Pipeline executes upstream meaning; it does not invent it.
2. **Visual Syntax First in Harness development.** Runtime cannot reinterpret specimen evidence or Category constitutions.
3. **Builder remains canonical.** No second Harness compiler or semantic IR is introduced.
4. **Brownfield before rewrite.** Existing CMF Studio runtime code is reused, adapted or activated when it fits current contracts.
5. **Workflow Nodes, not monolithic agents.** Responsibilities, owners, tools and handoffs remain explicit.
6. **Retrieval before reasoning.** Eligibility is deterministic; ranking is hybrid; execution receives a Minimum Complete Context.
7. **Stable competence in weights, current truth in context.** Programmed Models learn bounded behavior; mutable doctrine and evidence stay explicit.
8. **Editing taste is layered.** Law, Skills, Steering Recipes, learned judgment, evaluation and human authority remain distinct.
9. **Renderers realize approved programs.** Skia, Remotion, HyperFrames and FFmpeg cannot redefine creative intent.
10. **Verifiability over vibes.** Mechanical facts are mechanically proven; judgment is independently evaluated; human authority resolves the remainder.
11. **No uncontrolled online self-modification.** Production episodes become evidence; releases change behavior.
12. **Distinctive creative freedom is explicit.** Transformation Contracts define what must stay true and where invention is encouraged rather than steering toward a generic centroid.

# Success Metrics and Counter-Metrics

## Release 1 success metrics

- 100% of accepted runs verify exact Harness, binding, JIT Capsule and execution fingerprint identities.
- Zero constitutionally ineligible retrieval results enter a JIT Capsule.
- Zero out-of-contract mutations pass acceptance.
- The first Programmed Model claim materially improves its declared advantage profile over the approved general-model baseline.
- Expected candidates per accepted result decreases as the specialist matures.
- Skia outputs are actual deterministic artifacts with measured text, geometry and Visual Syntax reparse evidence.
- Local diagnosis repairs the intended failed unit without unnecessary regeneration.
- Replay and cancellation reproduce the declared result and history.

## Product metrics after expansion

- Acceptance rate by Category/profile/claim/runtime.
- Human review minutes per accepted asset.
- Candidate count, repair rounds, latency and cost per accepted result.
- Retrieval top-K coverage, contradiction coverage and downstream decision lift.
- Runtime-specific technical failure and fallback rates.
- Programmed Model precision, variance, tail failures and escalation quality.

## Counter-metrics

- Aesthetic polish with wrong Activative or Category reading.
- Lower cost achieved through unqualified runtime/model substitution.
- Fewer human reviews caused by hidden automation rather than improved precision.
- Benchmark gains achieved through leakage, weak verifiers or process violations.
- Higher throughput accompanied by lower acceptance or more repairs.
