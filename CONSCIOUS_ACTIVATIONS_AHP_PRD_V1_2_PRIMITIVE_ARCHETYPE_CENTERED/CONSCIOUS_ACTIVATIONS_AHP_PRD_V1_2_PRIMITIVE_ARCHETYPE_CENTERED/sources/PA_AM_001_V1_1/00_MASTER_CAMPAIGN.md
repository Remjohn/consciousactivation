## Mission

Activate the existing Conscious Activations repositories and predecessor CMF Studio runtime
into one usable production system.

The campaign does not require every Atomic Harness, every Programmed Model, every
provider, or full certification. It must leave a system in which new Harnesses,
execution bindings, runtime adapters, and Programmed Models can be added incrementally.

## Non-negotiable architecture

```text
Atomic Harness Builder
    compiles AtomicHarnessDefinition

Atomic Harness Pipeline
    validates and executes the definition

Existing Interview Expression product
    produces Complete Expression Sessions, Expression Moments,
    Expression Ingredient Inventories, and Asset Package Specs

Programmed Models
    implement bounded learned capability claims

Governed Retrieval
    compiles Minimum Complete Context into JIT Execution Capsules

CMF-derived runtimes
    execute composition and editing mechanics

Skia
    static composition

Remotion
    React/frame-based temporal composition

HyperFrames
    HTML/CSS/GSAP temporal composition

FFmpeg
    media editing, finishing, encoding, and technical verification

Visual Asset Editor
    produces, transforms, segments, mattes, evaluates, and repairs ingredients
    and owns the GNM main-avatar facial-geometry/expression-reference tool

Delegation
    carries typed work across independent product boundaries
```

## Product-boundary correction for GNM

GNM is not an Interview Expression subsystem and is not a Pipeline-core runtime.

```text
Desired avatar expression or approved visual-expression specification
        ↓
Atomic Harness Pipeline
        ↓
Visual Asset Demand through Delegation when crossing product ownership
        ↓
Visual Asset Editor
        ↓
GNM facial geometry and expression reference generation
        ↓
ComfyUI image generation or image editing
        ↓
Avatar Asset Result
```

Interview Expression may provide an `ExpressionMoment` or an `AssetPackageSpec` that
eventually motivates a Visual Asset Demand. It does not load GNM, generate GNM
coefficients, own GNM models, or require GNM to complete its own product workflow.

## Brownfield law

Do not reimplement a capability merely because its code is located in
`THE_CMF_STUDIO(2).zip`.

For every target component, issue one reuse disposition:

```text
REUSE_AS_IS

FORK_AND_REBIND
```

cannot satisfy the current contract economically.

## Launch floor

The campaign reaches limited production when all of these are real:

1. One current `AtomicHarnessDefinition` executes through the Pipeline bridge.
2. One Complete Expression Session can produce an Asset Package Spec through the
   existing Interview Expression product.
3. One static Harness produces real Skia artifacts.
4. One temporal Harness produces a real Remotion/FFmpeg artifact using the existing
   edit-decision and timeline contracts.
5. HyperFrames has one real bounded composition path or a fully validated executable
   adapter package with one exact external blocker.
6. Every successful result has real files, hashes, receipts, and operator acceptance.
7. Fake URIs and synthetic render references cannot satisfy the launch gate.
8. A new Harness can be added through an execution binding without changing Pipeline
   core semantics.

## Campaign execution waves

### Wave 0 — Two-hour canonical preflight

The integrator:

- verifies live local repository state rather than trusting stale public counts;
- locates and pins the existing Interview Expression product root;
- freezes source identities;
- extracts `THE_CMF_STUDIO(2).zip` read-only;
- assigns path ownership and exclusions;
- runs all product baselines;
- records exact READY existing Stories;
- finalizes only PRD defects that block implementation;
- creates shared campaign state.

No implementation lane waits for documentation cosmetics that do not affect contracts.

### Wave 1 — Parallel activation

Start Lanes A through G concurrently after path locks are issued.

### Wave 2 — Parallel vertical integrations

- Existing Interview Expression product → Asset Package Spec → Pipeline.
- Static Harness → Skia.
- Edited-video Harness → existing edit decisions → Remotion/FFmpeg.
- VAE Asset Result → Pipeline.
- VAE GNM reference pack → ComfyUI avatar generation/editing.
- Retrieval → JIT Capsule → reference executor.

The Interview Expression product and GNM remain independently executable. Their only
connection is an optional typed downstream asset demand.

### Wave 3 — Operator and release integration

The integrator serializes shared contracts, runs cross-product regressions, performs
real production trials, and cuts the limited-production release.

## Terminal campaign gate

Run:

- focused tests for every lane;
- affected predecessor suites;
- architecture boundaries;
- full regression for every modified product twice from fresh processes;
- real artifact existence and hash checks;
- replay, resume, and cancellation checks;
- portable-path and secret scans;
- fake-artifact rejection tests;
- dependency and license inventory;
- exact receipt and manifest reconciliation.

## Campaign stop law

A lane stops only for:

- missing external credentials or spend authority;
- unresolved third-party licensing;
- absent human truth or irreversible approval;
- source corruption;
- a genuine architecture contradiction that cannot be adapted.

A failed test, missing package, stale allowlist, or implementation defect is work to fix,
not a human product blocker.
