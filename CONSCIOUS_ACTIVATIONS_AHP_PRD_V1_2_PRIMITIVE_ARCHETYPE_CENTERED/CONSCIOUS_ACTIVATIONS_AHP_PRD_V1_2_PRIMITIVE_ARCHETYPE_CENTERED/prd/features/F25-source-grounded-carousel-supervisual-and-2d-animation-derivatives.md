# F25 — Source-Grounded Carousel, SuperVisual, and 2D Animation Derivatives

## 1. Product claim and user outcome

**User outcome:** Static and animated outputs become format-native embodiments of the same approved source expression.

Preserve source, evidence, quote/voice transformation labels, identity, and batch continuity across derivatives while keeping Format 02 deferred. This feature exists to convert doctrine and source evidence into behavior that can be implemented, tested, denied, replayed, and audited. It does not grant implementation, provider, model-training, publication, production, or certification authority by itself.

## 2. Product boundary and ownership

Pipeline owns derivative programs; VAE owns missing visual ingredients; runtime adapters render.

The feature must preserve the current product topology. Builder supplies the immutable Atomic Harness; Pipeline executes its declared nodes; Interview Expression owns source expression preparation and approval; VAE owns visual production; Delegation owns cross-product envelopes; Studio projects state and emits typed operator commands; Program Control owns cross-product authority and release truth. External renderers and models are embodiments, not sources of meaning.

## 3. Canonical objects and state

**Cross-cutting Activative law:** Content activates when it gives the viewer a psychological role inside a tension. Content-bearing work inherits the approved archetype coalition, Primitive Coalition Contract, Final Script, Source Fidelity, Negative Space, Edge Integrity / Anti-Centroid, and Brand/Voice/Visual context when applicable.

The feature consumes current, versioned upstream records and emits typed outputs that remain linked to the active Harness, source package where applicable, Workflow Node, JIT Capsule, execution binding, evaluator, and receipt. Canonical records use stable serialization and immutable identities. A UI projection, model response, provider job, renderer workspace, or local cache cannot become accepted product state until its contract and authority gates pass.

Core invariants for this feature are:

- Derivative form does not erase source lineage.
- Quotes and generated voice are labeled truthfully.
- Animation does not activate Format 02 by implication.

## 5. Workflow integration

Approved source ingredients → format-native composition/animation program → deterministic or bound rendering → cross-derivative continuity → lineage export.

Every handoff is typed. A downstream node begins only after the producer output passes the declared contract and evidence gates. Failures are attributed to source, knowledge, retrieval, Skill, recipe, Programmed Model, tool, runtime, product, evaluator, or human-policy layers before repair or escalation.

## 6. Functional Requirements

### FR-145 — Produce source-grounded Carousel derivatives

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** A Carousel job shall derive slide claims, exact quotes, evidence, keyframes, visual references, and sequence roles from the accepted source package and approved archetype route, preserving source links for every slide while using Carousel-native swipe grammar.

**Trigger and preconditions:** An approved source package and Carousel-capable Harness/profile are available.

**Required output or state transition:** Carousel Composition IR, slide-role plan, source/evidence map, rendered images, preview, and evaluation.

**Authority and invariants:**
- Slides are not frame-time video scenes.
- Editorial paraphrases are labeled.
- Every source-backed assertion resolves to evidence.

**Acceptance scenarios**

- **Primary success:** Given an approved five-slide teaching route, when rendered, then each slide links to the supporting quote/evidence and the sequence preserves progression.
- **Adversarial or denial case:** Given a quote slide whose text cannot be found in the source and is not labeled paraphrase, when validation runs, then it fails.
- **Evidence required:** Slide/source traceability, geometry/text checks, sequence evaluation, and render hashes.

**Governing decisions:** D044, D046  
**Exact source references:** SRC-INT-001, SRC-CUR-007, SRC-LEG-024, SRC-LEG-025


### FR-146 — Produce source-grounded SuperVisual derivatives

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** A SuperVisual job shall condense one source-backed premise, contradiction, identity statement, proof, or participation invitation into one feed-legible visual argument with exact source/evidence lineage and single-frame composition laws.

**Trigger and preconditions:** An approved source moment and SuperVisual-capable Harness/profile are available.

**Required output or state transition:** Single-frame Composition IR, source/evidence map, master image, variants, and evaluation.

**Authority and invariants:**
- One frame cannot imply unsupported meaning.
- Visual hierarchy is measurable.
- Source quote and editorial language remain distinguishable.

**Acceptance scenarios**

- **Primary success:** Given an approved premise and portrait keyframe, when rendered, then the visual preserves identity prominence and cites its source.
- **Adversarial or denial case:** Given a beautiful poster whose slogan overstates the speaker's claim, when evaluated, then it fails semantic fidelity.
- **Evidence required:** Source map, feed-size legibility, reading-order evidence, semantic evaluation, and hashes.

**Governing decisions:** D044, D046  
**Exact source references:** SRC-INT-001, SRC-CUR-007, SRC-LEG-026, SRC-LEG-027


### FR-147 — Produce source-grounded 2D animation derivatives without activating Format 02

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** An interview-derived 2D animation short may use source audio, approved voiceover derivatives, quotes, evidence, keyframes, identity references, and expression/gesture cues through a separately authorized animation Harness and runtime, but it shall not be labeled or implemented as Format 02 unless the Format 02 activation gate is later satisfied.

**Trigger and preconditions:** An authorized non-Format-02 animation derivative job is admitted.

**Required output or state transition:** Animation scene/clip program, source/voice lineage, rig/layer/runtime bindings, rendered output, and evaluation.

**Authority and invariants:**
- Format 02 remains deferred.
- Generated performance does not claim to be the participant's actual reaction unless directly derived and labeled.
- Identity references are governed.

**Acceptance scenarios**

- **Primary success:** Given a quote-driven animated explainer using the interviewer's audio, when produced, then every spoken segment and visual idea traces to the source or an explicitly approved derivative.
- **Adversarial or denial case:** Given a Stretchy Studio rig demo, when no current animation Harness authorizes its use, then it cannot enter production.
- **Evidence required:** Animation-job authority, source/voice map, runtime/rig receipts, identity/semantic evaluation, and Format 02 exclusion test.

**Governing decisions:** D044, D051, D053  
**Exact source references:** SRC-EXT-019, SRC-EXT-025, SRC-AM-001, SRC-INT-001


### FR-148 — Label quotations, condensations, voiceovers, and generated derivatives truthfully

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** Every derivative shall distinguish verbatim source quote, edited quote with disclosed omission, faithful condensation, operator-authored bridge, generated voiceover, synthetic voice, and source audio, preserving speaker attribution, approval, and transformation lineage.

**Trigger and preconditions:** A derivative transforms or reuses spoken language or voice.

**Required output or state transition:** Typed language/voice provenance records attached to content elements and exports.

**Authority and invariants:**
- No paraphrase is presented as verbatim.
- Synthetic or rerecorded voice is clearly identified in records.
- Operator source authorization and identity use govern generation.

**Acceptance scenarios**

- **Primary success:** Given a Carousel condensation of a long answer, when displayed, then it is labeled editorial condensation and links to the source span.
- **Adversarial or denial case:** Given a cloned voice reading newly written text without approved synthetic-voice operator source authorization, when production is attempted, then it is denied.
- **Evidence required:** Quote-diff checks, voice/source metadata, operator source authorization validation, operator approval, and export disclosures.

**Governing decisions:** D044, D047  
**Exact source references:** SRC-INT-001, SRC-AM-001


### FR-149 — Preserve cross-derivative identity, meaning, and batch continuity

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** The batch shall evaluate whether shorts, Carousels, SuperVisuals, and animation derivatives remain coherent in identity, terminology, evidence, visual recurrence, and intended audience movement while allowing format-native variation.

**Trigger and preconditions:** Two or more derivative jobs from the same source package reach candidate or accepted state.

**Required output or state transition:** A batch continuity report, shared/repeated element map, contradictions, fatigue/recurrence findings, and repair recommendations.

**Authority and invariants:**
- Continuity does not require visual sameness.
- Category-native grammar remains independent.
- Contradictions are not hidden.

**Acceptance scenarios**

- **Primary success:** Given a short and Carousel using the same premise, when compared, then their core claim and source evidence agree while each uses its own format grammar.
- **Adversarial or denial case:** Given a SuperVisual that reverses the short's call or misattributes a quote, when batch evaluation runs, then acceptance is blocked.
- **Evidence required:** Cross-artifact semantic comparison, identity/source checks, recurrence analysis, and batch verdict.

**Governing decisions:** D044, D046  
**Exact source references:** SRC-CUR-020, SRC-INT-001


### FR-150 — Export complete derivative lineage and usage evidence

**Lifecycle:** `NEW_V1_1`  
**Normative requirement:** Every derivative output shall package source-package and span references, ingredient versions, Harness and binding identities, transformation records, generated-asset lineage, evaluations, human resolutions, publication scope, and downstream usage acknowledgements.

**Trigger and preconditions:** A derivative reaches terminal accepted, rejected, canceled, shadow, or published state.

**Required output or state transition:** A portable derivative evidence bundle and usage records.

**Authority and invariants:**
- No accepted output is orphaned from source.
- Restrictions and limitations travel with exports.
- Historical versions remain addressable.

**Acceptance scenarios**

- **Primary success:** Given a published Carousel, when audited, then every slide's source/evidence and every generated visual's VAE lineage can be traced.
- **Adversarial or denial case:** Given an exported animation with missing voice provenance, when package validation runs, then delivery fails.
- **Evidence required:** Lineage completeness test, artifact hashes, usage acknowledgement, and portable bundle validation.

**Governing decisions:** D044, D047  
**Exact source references:** SRC-CUR-022, SRC-AM-001


## 7. Execution, model, tool, and human responsibilities

Deterministic services own mechanically decidable facts. Programmed Models or bounded Agent Programs may propose or compile behavior only within their evidence-backed claim and node-local tool grants. Independent evaluators inspect judgment dimensions. Humans own new meaning, identity interpretation, unresolved taste, operator source authority, promotion, and publication authority where assigned. The producing model or runtime cannot accept its own output.

When the feature uses a Hunter, Analyst, Composer, or Commander role, the role is an explicit workflow responsibility. It may be implemented by code, a small model, a larger model, or a human. The role name never hides the actual actor or expand its authority.

## 8. Validation and quality gates

Validation must prove both the intended result and the process that produced it. The minimum feature evidence includes source maps, static/animation renders, quote/voice provenance, batch continuity, usage and export bundles. Mechanical tests run before expensive model or visual evaluation. Judgment evaluators are calibrated to the exact category and context. Every denial names the cause, owner, and next admissible action. Every accepted mutation emits an immutable receipt and updates only dependent state.

## 9. Risks, mitigations, and stop conditions

| Risk | Concrete consequence | Mitigation |
|---|---|---|
| Authority drift | A predecessor, external project, UI, model, or renderer is treated as current product authority. | Require exact source disposition, current authority validation, and product-boundary tests. |
| False completion | A synthetic URI, dry-run, unverified command, or model self-report is promoted as a real result. | Require actual files, hashes, independent validation, and artifact-class labels. |
| Over-broad automation | A model or agent gains tools, context, or authority beyond the current node. | Use JIT context, node-local tool grants, sandboxes, and explicit escalation. |
| Hidden mutation | Accepted meaning, source, program, or human preference changes without a versioned record. | Use immutable versions, typed commands, receipts, and selective invalidation. |
| Source truth loss | A derived artifact, route, quote, or cut weakens or changes the participant's actual expression. | Require exact source context, quote/voice labels, independent semantic review, and downstream lineage. |

**Stop conditions**

- The behavior would create a second semantic or canonical state owner.
- A required source, authority, contract, primitive coalition, archetype, Final Script, runtime, evaluator, or human decision is unavailable or unverified.
- The implementation would treat migration evidence or an external project as current authority.
- The change would imply real output, production readiness, or certification beyond available evidence.
