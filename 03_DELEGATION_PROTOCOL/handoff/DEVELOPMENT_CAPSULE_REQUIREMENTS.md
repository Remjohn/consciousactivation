# Delegation Development Capsule Requirements

Implementation authorization requires a versioned capsule containing:

## Product and governance

- approved sharded PRD
- Decision Register
- Requirements Registry and Traceability Matrix
- Protocol Constitution
- Authority Matrix and Principal Registry
- Lifecycle Machine
- Failure Taxonomy
- Compatibility Policy
- Architecture Preservation Contract
- Readiness Hard Gates

## Architecture

- Architecture document and all ADRs
- component and deployment diagrams
- data, message and event models
- security and integrity design
- storage, availability and recovery design
- Control Tower projection design

## Contracts

- final shared schema family
- message registry
- positive and negative fixtures
- compatibility manifests
- adapters and migrations
- Format 02 cross-product fixtures

## Verification

- producer and consumer contract tests
- authority and lifecycle suites
- compatibility and migration suites
- resilience/fault-injection suites
- audit-chain and replay tests
- Format 02 end-to-end conformance
- SLO measurement plan

## Planning

- approved outcome-centered epics
- dependency-safe vertical stories
- feature technical specifications
- implementation traceability
- migration and rollback plan
- implementation authorization receipt

No implementation team may invent shared authority, lifecycle, failure or compatibility semantics outside this capsule.

The capsule must pin the constitution-complete Visual Asset Demand `1.1`, the
exact lineage identity shape, the owner-evidenced V1-to-V1.1 migration, and the
five constitutional conformance cases. It must carry evidence that non-empty
wrong-reading locks are enforced/evaluated and that parse-only or lossy
compatibility is rejected before admission.
