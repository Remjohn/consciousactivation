# Cross-Repository Release Order

1. Delegation publishes a canonical amended contract release.
2. VAE pins the release and proves intake/result compatibility.
3. Builder aligns target profiles and compilers to the pinned contract profile.
4. The program runs one cross-repository convergence gate.
5. Implementation proceeds only where local readiness permits it.

Repositories exchange release artifacts and status exports. Agents do not edit one another's active folders.
