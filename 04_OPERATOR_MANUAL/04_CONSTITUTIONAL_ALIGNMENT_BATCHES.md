# Constitutional Alignment Batches

## Batch A — Authority and requirements
Map the amendment. Do not patch broad artifacts yet.

## Batch B — Contracts and schemas
Change shared structures and compatibility behavior.

## Batch C — Technical specifications and ADRs
Patch only implementation design affected by A/B.

## Batch D — Planning and readiness
Update epics, stories, blockers and authorization evidence.

## Batch E — Implementation and tests
Allowed only under the repository's current implementation authority.

Each batch ends with a validation report, status export and explicit verdict.
